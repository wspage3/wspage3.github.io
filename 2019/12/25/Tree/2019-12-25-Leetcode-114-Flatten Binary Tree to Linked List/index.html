【二叉树专项】114.Flatten Binary Tree to Linked List

一、题意

题目链接: 114.Flatten Binary Tree to Linked List

1.Description

【输入】
给定一棵二叉树。
【输出】
使其扁平化，退化成一个单链表。
要求【in-place】操作：此处的in-place，应该指的是在当前的节点上改变指向。而非严格O(1)的空间复杂度。
单链表的顺序为：preorder遍历树的顺序。

2.Example

Input: [1,2,5,3,4,NULL,6]
Output: [1,2,3,4,5,6]

3.Corner Case

  1. 树为NULL时，无需任何操作，return即可。

二、题解

分析

目的：使得树退化为单链表，同时单链表从前到后的顺序为：前序遍历该树的顺序。即：根 -> 左 -> 右
方法：使用相反的顺序遍历树（右 -> 左 -> 根）：每遍历一个节点，将其链接到已退化好的单链表中，更新单链表的头。

Solution-1：Recursive Solution(DFS-Postorder Traversal)

1.回忆【145.Binary Tree Postorder Traversal】的迭代版本：
我们要想得到：左 -> 右 -> 根这样的后序遍历序列。
采用的遍历顺序实际上是：根 -> 右 -> 左。最后将遍历得到的序列进行反转。

2.在本题中，采用类似的思路：
对于任一节点：
首先访问其右儿子，然后左儿子，最后访问自己。可以得到：右 -> 左 -> 根这样的后序遍历序列。
访问每个节点：将其右儿子赋值为单链表的头。将其左儿子赋值为空。将单链表的头设置为当前节点。

3.这样的访问顺序，使得最后一个访问的节点为root。
访问完root，root指针就指向一个单链表了。

时间复杂度：O(n)
空间复杂度：O(n)

4.Code(https://leetcode.com/submissions/detail/278369749/)

Solution-2：Iterative Solution

1.对于每一个节点cur，其左子树cur->left，其右子树cur->right。
考虑cur->left不为空的情况：
想办法将这两个儿子合并，成为一棵树。
这样的话，对于cur来说，就满足了链表节点的性质：只有一个后继节点。

2.由于答案顺序为：根 -> 左 -> 右
所以将右子树(cur->right)拼接到左子树(cur->left)的“最后一个”节点即可。
对于cur->left来说，“最后一个”节点即：cur->left的第一个为空的右儿子。

时间复杂度：O(n)
空间复杂度：O(1)
