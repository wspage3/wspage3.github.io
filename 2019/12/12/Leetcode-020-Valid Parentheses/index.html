20.Valid Parentheses

发表于 2019-12-12

一、题意

题目链接: 20.Valid Parentheses

1.Description

给定一个长度为n的字符串s；
判断该字符串是否一个合法的括号表达式；
括号有三种：[] {} ()
open的bracket最终必须有一个同类型close的bracket与之匹配；
且上述匹配的顺序必须是正确的。

2.Example

Input: “()[]{}”
Output: true

Input: “([)]”
Output: false

3.Corner Case

n: 当n为0的时候，也认为是合法的；

二、题解

Solution-1：Stack

1.若n是奇数，那么一定非法。
若n是0，则直接return true。

2.考虑n为大于0的偶数：
模拟括号匹配的过程：i在范围[0, n - 1]内遍历：
先将s[0]压入栈；
考察s[1]，若s[1]和s[0]是同类型的，且一个s[1]是close，s[0]是open的，那么说明这两个是一对。
则此时将s[0]从栈中抛出，i跳到位置2，考虑第二个元素。
若s[1]的出现无法和s[0]匹配掉，则将s[1]在压入栈，考虑下一个元素。
……
直到所有的n个元素都遍历过了，此时检查栈，如果栈是空的，说明所有的括号都可以匹配。
如果此时栈非空，则说明存在元素无法匹配。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/285447709/)

Solution-2：Stack

1.另一种方法：
从[0, n - 1]考虑每一个s[i]
若s[i]是open的bracket，则将其“另一半”压入栈。
否则的话，判断此时栈是否为空，是的话，直接return false。
栈不空的话，判断当前元素是否和栈顶元素相同，不同的话，也直接return false。
相同的话，将栈顶元素抛出，转移到考虑s[i + 1]

时间复杂度：O(n)
空间复杂度：O(n)
