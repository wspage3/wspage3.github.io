34.Find First and Last Position of Element in Sorted Array

发表于 2019-12-12 更新于 2019-12-13

一、题意

题目链接: 34.Find First and Last Position of Element in Sorted Array

1.Description

给定一个长度为n的数组，该数组是升序排列的，中间可能有重复的元素；
给定一个整数target；
查找target是否在数组中，在的话，返回其开始出现的索引，以及最后一次出现的索引。
不存在的话，返回(-1, -1)即可；

2.Example

Input: nums = [5,7,7,8,8,10], target = 8
Output: [3,4]

Input: nums = [5,7,7,8,8,10], target = 6
Output: [-1,-1]

3.Corner Case

n: 当n为0的时候，直接返回(-1, -1)；

二、题解

Solution-1：Binary Search + Linear Search

1.二分查找，每次去mid，然后判断nums[mid]的值是否是target，如果是的话：
依次向前找，直到当前的数不等于target为止。
依次向后找，直到当前的数不等于target位置。
找到，并return 即可。
如果target大于nums[mid]，则在[mid + 1, right]中搜。
如果target小于nums[mid]，则在[left, mid - 1]中搜。

2.特殊情况考虑：当left，right相差1的时候，满足while循环条件，并进入。
此时mid = left，考虑了nums[mid]后:
若nums[mid]就是我们要找的target，则寻找它开始结束的位置。
若target大于nums[mid]，此时left = right，while循环退出。此时需要考虑left(等于right)这个位置的值。
如果是target，则返回{left, left}即可。
如果不是，则target不在数组中。
若target小于nums[mid]，则target不在数组中。

时间复杂度：O(log n)，最坏O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/285482172/)

Solution-2：Binary Search + Binary Search

1.由于我们在Leetcode-704-Binary Search中学会了找target的最小索引和最大索引。
正好本题目中就是找这两个值。

2.先找到target的最小索引；再找到target的最大索引。
再返回答案。

时间复杂度：O(log n)
空间复杂度：O(1)
