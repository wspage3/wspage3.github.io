494.Target Sum

一、题意

题目链接: 494.Target Sum

1.Description

【输入】
给定长度为n的整数数组。数组元素都是非负数。
给定一个整数S。
【输出】
同时你拥有若干个加号，若干个减号。它们加起来为n。
把这n个加号或者减号放在nums[i]的前面，使其组成一个加减法表达式。
求出存在多少个不同的表达式，使得表达式的值为target。

2.Example

Input: nums is [1, 1, 1, 1, 1], S is 3.
Output: 5

Explanation:
-1+1+1+1+1 = 3
+1-1+1+1+1 = 3
+1+1-1+1+1 = 3
+1+1+1-1+1 = 3
+1+1+1+1-1 = 3
There are 5 ways to assign symbols to make the sum of nums be target 3.

3.Corner Case

  1. 0 < n <= 20
  2. nums数组累加和 <= 1000
  3. 返回结果在32位整数表示范围内

二、题解

Solution-1：Brute Force

1.由于需要在n个位置放置，每个位置只能放置’+’或者’-‘。
所以一共有(2 ^ n)种不同的放置方案。
枚举所有的情况，并计算其表达式的值，统计等于target的次数。

2.使用递归来简化这个过程：
从i = 0出发，当前累加sum为0。
若当前i小于等于n - 1，说明所有元素还未全部考虑完成。
若当前i等于n，说明所有元素都考虑完成了，统计此时累加的sum，如果和S相等，则统计一次。

对于位置i，在sum的基础上可以继续累加(+nums[i])，也可以累加(-nums[i])。即：
dfs(i + 1, target + nums[i]);
dfs(i + 1, target - nums[i]);

时间复杂度：O(2 ^ N)
空间复杂度：O(N) //递归栈的开销

6.Code(https://leetcode.com/submissions/detail/287342127/)

Solution-2：Dynamic Programming

1.dp[i][j]表示：区间[0, i - 1]为止，任意组合使得和为j的方案数。
i取值范围: [0, n - 1]
j取值范围: [-1000, 1000]
状态转移方程：dp[i][j] = dp[i - 1][j - nums[i]] + dp[i - 1][j + nums[i]]
由于数组index不支持负数。我们把j的区间向右平移：1000。j取值范围: [0, 2000]。
对应情况：
旧index：[-1000, -999, -998, …, -2, -1, 0, 1, 2, …, 999, 1000]
新index：[0, 1, 2, …, 998, 999, 1000, 1001, 1002, …, 1999, 2000]

2.边界值情况：
当i为0的时候，记temp = nums[0]。
旧index：dp[0][-temp] = 1, dp[0][temp] = 1
新index：dp[0][-temp + 1000] = 1, dp[0][temp + 1000] = 1
注意：此时若temp为0的话，分为两种情况：-0，+0。
所以应该写成：dp[0][-temp + 1000]++, dp[0][temp + 1000]++

3.过程：
i从1开始，遍历到n - 1为止。
j从0开始，遍历到2000为止。
dp[i][j] = dp[i - 1][j - nums[i]] + dp[i - 1][j + nums[i]]

注意1：此时(j - nums[i])和(j + nums[i])代表的是：一种相对距离。
举个例子：当i为5，num[i]为20，j为1050的时候：
dp[5][1050] = dp[4][1050 - 20] + dp[4][1050 + 20]
由于存在index后移的情况，原始转移方程：
dp[5][50] = dp[4][30] + dp[4][70]

注意2：
j - nums[i]可能小于0
j + nums[i]可能大于2000
超过合法范围，均无意义。即为0。

4.结果返回dp[n - 1][S + 1000]即可。
此时若S已经大于了1000，说明就算所有数字选择‘+’，和也不可能达到S。返回0即可。

5.优化1：不必固定认为和的最大值为1000，每次计算一下，然后再申请数组即可。

6.优化2：空间压缩。二维空间压缩为一维空间。

时间复杂度：O(N * sum)
空间复杂度：O(sum) //sum为所有nums[i]的和
