【DP专项】416.Partition Equal Subset Sum

一、题意

题目链接: 416.Partition Equal Subset Sum

1.Description

【输入】
给定一个长度为n的整数数组。n大于0。所有的元素都是正整数。
【输出】
判断这个数组中是否能分成两个数组，它们的和相等。

2.Example

Input: [1, 5, 11, 5]
Output: true
Explanation: The array can be partitioned as [1, 5, 5] and [11].

3.Corner Case

  1. n: 大于0
  2. nums[i]: 大于0

二、题解

思路

  1. 实质上：本题目就是求nums数组的一个子序列（不要求连续），使得其和为nums数组的和的一半。

  2. 求出nums数组的和sum。若sum是奇数，则直接return false。

  3. 回忆之前做过的【从数组中选一些值，使得sum为target】题目，均使用DP解法：
    a. 【377.Combination Sum IV】：从给定数组（元素不重复）中选出一些元素，使得其sum为target，问一共有多少种方案？【每个元素可选可不选】【每个元素可选多次】【21-12是两种方案】
    b. 【322.Coin Change】：从给定数组（元素不重复）中选出一些元素，使得其sum为target，问最少使用几个数字？【每个元素可选可不选】【每个元素可选多次】【21-12是两种方案，但本题求min值不关心】
    c. 【518.Coin Change 2】：从给定数组（元素不重复）中选出一些元素，使得其sum为target，问一共有多少种方案？【每个元素可选可不选】【每个元素可选多次】【有序遍历nums[i], 21-12只有其中之一会遍历到, 故不会重复计数】
    c. 【494.Target Sum】：从给定数组（元素可重复）中，每个元素选+或者-，使得其sum为target，问一共有多少种方案？【每个元素必选】【每个元素可选1次】【有序遍历nums[i]】

  4. 本题特点：从给定数组（元素可重复）中选出一些元素，使得其sum为target，问是否有解？【每个元素可选可不选】【每个元素可选1次】==>【有序遍历nums[i]】

Solution-1：Dynamic Programming

  1. 状态定义：dp[i][j]代表：[0, i]范围内是否可以使得和为j。bool类型即可。

  2. 边界条件：
    a. 不管i为多少，若j为0，则dp[i][0] = true；（为方便后续计算）如：nums:[3,1,6,2] target = 6。当i = 2，j = 6时，dp[2][6] = dp[1][6] || dp[1][0] = false || dp[1][0]。
    b. 当i为0时，j == nums[0]的时候，dp[0][j] = true；

  3. 转移方程：dp[i][j] = (dp[i - 1][j]) || (dp[i - 1][j - nums[i]])
    左边代表：不取nums[i]
    右边代表：选择nums[i]。当nums[i] > j的时候，右边值为false。

  4. 计算顺序：
    【Backward-DP】
    从nums[1]开始，到nums[n - 1]结束；
    对于每个nums[i]，考虑其出现，对于所有可能的和（0-target）的影响。

  5. 空间压缩：二维空间降低到一维空间。

  6. Code(https://leetcode.com/submissions/detail/286289354/)
    时间复杂度：O(n * c) //c代表nums数组累加和的一半
    空间复杂度：O(n * c)

  7. 进一步，压缩空间复杂度：
    从二维的方程可以看出，当前位置的值依赖于两个值：这个位置上方的，这个位置左上方的。
    如果压缩为一维的时候，继续让j从小到大遍历的话，这样当需要‘左上方’的时候，这个值一定是被覆盖过的。
    所以需要让j从大到小遍历。

  8. Code(https://leetcode.com/submissions/detail/286325812/)
    时间复杂度：O(n * c) //c代表nums数组累加和的一半
    空间复杂度：O(c)

Solution-2：DFS

  1. 将数组由大到小排序。

  2. return dfs(nums, sum / 2, 0);

  3. 在递归中：
    首先判断index是否在[0, n - 1]范围内，若超过该范围，return false。
    然后判断nums[index] == target，如果满足，则return true。
    接着判断nums[index] > target，如果满足，则return false。
    最后递归调用左子树和右子树即可，返回它们的返回值的or运算结果。
    左子树：dfs(nums, target - nums[index], index + 1);
    右子树：dfs(nums, target, index + 1);

  4. 如果先调用右子树的话，会TLE。
