153.Find Minimum in Rotated Sorted Array

发表于 2019-12-11

一、题意

题目链接: 153.Find Minimum in Rotated Sorted Array

1.Description

给定一个长度为n的整数数组nums；
该数组原本是升序的，结果在某个支点处旋转了一次，如：
i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]
假设数组中不存在相同的数字。
找出这个数组中最小的值。

2.Example

Input: [3,4,5,1,2]
Output: 1

3.Corner Case

n: 当n为0的时候，直接返回false；

二、题解

Solution-1：Brute Force

1.常规思路：从前到后遍历一遍nums，找到最小值返回。

时间复杂度：O(n)
空间复杂度：O(1)

Solution-2：Binary Search

1.观察：[0,1,2,4,5,6]
经过rotate后，可能存在以下几种不同情况：
[1,2,4,5,6, 0]
[2,4,5,6, 0,1]
[4,5,6, 0,1,2]
[5,6, 0,1,2,4]
[6, 0,1,2,4,5]

2.考虑寻找[left, right]中的最小值。
若：nums[left] < nums[right]，则说明[left, right]范围内完全升序。nums[left]就是答案。
否则：
mid = left + (right - left) / 2;
若nums[mid] > nums[right]，则说明，这个区间中间某个值比nums[right]还要大，说明mid此时在左侧。
而最小值一定在右侧[mid + 1, right]。
若nums[mid] < nums[right]，则说明，[mid, right]这个范围是有序的。要寻找的最小值，不可能在mid右侧，一定在包含其的左侧，即：[left, mid]。
若nums[mid] == nums[right]，不可能的……

3.用二分的方法，每次把结果可能出现的区间缩小一半。
最终当区间长度为2的时候，left = x, right = x + 1, mid = x
判断nums[mid]和nums[right]的大小关系，也就是判断左边元素和右边元素的大小。
若nums[mid]大于nums[right]，即答案一定在[mid + 1, right]中，也就是右边元素本身。
若nums[mid]小于nums[right]，即答案一定在[left, mid]中，也就是左边元素本身。
循环结束后，left一定等于right。此时返回left的值即可。

时间复杂度：O(log n)
空间复杂度：O(1)
