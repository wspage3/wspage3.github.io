81. Search in Rotated Sorted Array II

发表于 2019-12-11

一、题意

题目链接: 81. Search in Rotated Sorted Array II

1.Description

给定一个长度为n的整数数组nums；
该数组原本是升序的，结果在某个支点处旋转了一次，如：
i.e., [0,0,1,2,2,5,6] might become [2,5,6,0,0,1,2]
给定一个整数target；
判断nums中是否包含该target，如果是，返回ture；否则，返回false。
假设数组中存在相同的数字。

2.Example

Input: nums = [2,5,6,0,0,1,2], target = 0
Output: true

Input: nums = [2,5,6,0,0,1,2], target = 3
Output: false

3.Corner Case

n: 当n为0的时候，直接返回false；

二、题解

Solution-1：Brute Force

1.常规思路：从前到后遍历一遍nums，如果找到target则return true。

时间复杂度：O(n)
空间复杂度：O(1)

Solution-2：Binary Search

1.参考leetcode-33的二分解法，只不过本题目中input可以存在相同的数字。
上题中核心思路就是：通过判断nums[mid]和target是否在同一侧，来确定比较器num。
前提是：右侧所有的元素都比nums[0]要小。
所以可以通过(nums[mid] < nums[0]) == (target < nums[0])来判断。
若其在同一侧，则使用nums[mid]作为比较器即可。
若其不在同一侧，则对应的将left右移或者right左移。继续考察下一次……

2.而本题目：右侧所有元素 <= nums[0]
假设把其所有和nums[0]相等的元素，都去掉的话，并不影响查找target；
这样的话，右边所有的数字都小于nums[0]了，继续使用leetcode-33的二分解法即可。

3.观察：[0,0,1,2,2,5,6]
经过rotate后，可能存在以下几种不同情况：
[0,1,2,2,5,6, 0]
[1,2,2,5,6, 0,0]
[2,2,5,6, 0,0,1]
[2,5,6, 0,0,1,2]
[5,6, 0,0,1,2,2]
[6, 0,0,1,2,2,5]

时间复杂度：平均O(log n) 最坏O(n)
空间复杂度：O(1)
