【二叉树专项】257.Binary Tree Paths

发表于 2019-12-26

一、题意

题目链接: 257.Binary Tree Paths

1.Description

【输入】
给定一棵二叉树。
【输出】
输出这棵二叉树中所有从root到leaf的路径。
每一条路径是一个字符串，最终返回一个一维字符串数组。
格式举例：”1->2->5”

2.Example

Input: [1,2,3,NULL,5]
Output: [“1->2->5”, “1->3”]
Explanation: All root-to-leaf paths are: 1->2->5, 1->3

3.Corner Case

  1. 当树为NULL时，返回一个空数组。

二、题解

Solution-1：Recursive Solution(Backtracking)

1.回忆【113.Path Sum II】，我们找出所有从root到leaf的路径，其和为target，保存在二维数组中。

2.现在我们要找出所有从root到leaf的路径，保存在一维字符串数组中。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275898943/)

Solution-2：Iterative Solution using Stack(DFS)

1.遍历顺序为迭代版本的DFS(preorder)

2.使用一个辅助栈，用于存放每个节点对应的字符串。
当用栈模拟节点的遍历时，同时也用辅助栈模拟其value。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275902870/)

【二叉树专项】236.Lowest Common Ancestor of a Binary Tree

发表于 2019-12-26

一、题意

题目链接: 236.Lowest Common Ancestor of a Binary Tree

1.Description

【输入】
给定一棵二叉树。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4
Output: 5
Explanation: The LCA of nodes 5 and 4 is 5, since a node can be a descendant of itself according to the LCA definition.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

思路

本题不同于上一题，上一题中输入的树为BST，可以通过比较节点的数值，每遍历一个节点，将问题规模缩小到原来的一半。

Solution-1：Recursive Solution

1.如果root为NULL，则return NULL

2.如果p,q中一者为root，则它俩的LCA一定为root

3.否则的话，说明：p和q一定是root的某一个子节点，分别在root的左子树和右子树中查找p,q的LCA
a.若在某个子树中找不到p和q，则代表p和q同时在本子树的兄弟子树中，此时返回NULL
b.若在某个子树中找到了p（或者q），则函数立马返回p（或者q）。
相当于说：某个后代M发现了两个目标（p或q）中的其中一个（例如p），立马报告自己的父亲A。
由于A还有一个儿子N，当M和N都给A汇报了结果后，A再进行决策。
a.若此时N也发现了目标中的另外一个，则A就是p和q的LCA。
b.若此时N没有发现目标，则A上报他自己的父亲B：p是属于以A为根的子树中的某个节点。
c.B收到这个信息，同时再看B的另外一个儿子的上报信息，决策并往上上报信息，直至root节点。
d.假设B在root的左子树中，root一看，发现p在其左子树中。同时root收到了其右子树的信息。
假设右子树中发现了q，此时root就可以说自己是p和q的LCA
假设右子树中发现了p（不可能的情况，p已在root的左子树中被发现了）
假设右子树中没有发现p,q的任何一个，即返回值为NULL。此时root一想，p已被发现，而q没有，那么p一定就是二者的LCA，则p为答案。
因为我们在发现了二者中的任意一个的时候，都立马返回，上报给父亲。而访问的顺序是dfs，说明未被发现的q一定为p的某个后代，故此p就是答案。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277036953/)

Solution-2：Iterative Solution

1.思路2：
找出从root到p的路径，存放在vectorP中：root, a, b, c, …, p
找出从root到q的路径，存放在vectorP中：root, a, b, c, …, q
从前往后对比两个vector，最后一个匹配的节点，即是答案。

2.在找从root到p、q的路径时，用dfs(preorder)的顺序：
遍历到某个节点，将其加入路径中。
如果这个节点为p或者q其中一个，则一条路径寻找完成。直到两条路径都找到，此时结束遍历。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277045028/)

【BST专项】235.Lowest Common Ancestor of a Binary Search Tree

发表于 2019-12-26

一、题意

题目链接: 235.Lowest Common Ancestor of a Binary Search Tree

1.Description

【输入】
给定一棵BST。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [6,2,8,0,4,7,9,null,null,3,5], p = 2, q = 8
Output: 6
Explanation: The LCA of nodes 2 and 8 is 6.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

Solution-1：Recursive Solution

1.思路：根据BST的性质：对于任意node来说，左子树中的所有节点都小于node的值；右子树中的所有节点都大于node的值；

2.具体方法：
从root节点开始，若p和q的val均小于root的val，则结果一定在root的左子树中
若p和q的val均大于root的val，则结果一定在root的右子树中
若一个大于root的val，一个小于，则pq的LCA一定为root

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276781092/)

Solution-2：Iterative Solution

1.减少递归栈的space complexity

2.具体方法：
每次修改root节点，将其赋值为其左儿子或者其右儿子。
缺点：修改了原始数据，使用前需确认评估。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/276783571/)

【BST专项】230.Kth Smallest Element in a BST

发表于 2019-12-26

一、题意

题目链接: 230.Kth Smallest Element in a BST

1.Description

【输入】
给定一棵BST。
给定一个整数k。（题目保证k合法，即 1 <= k <= n，n为BST的节点数）
【输出】
求出这棵BST中第k小的元素的值，以int形式返回。

2.Example

Input: [3,1,4,null,2], k = 1
Output: 1

3.Corner Case

  1. 树为NULL时，直接return 0。

二、题解

Solution-1：Iterative Solution(DFS-Inorder)

1.对BST进行中序遍历，并累加cnt当前节点是遍历的第几个节点。若cnt == k，则该节点就是答案。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276741636/)

【BST专项】501.Find Mode in Binary Search Tree

发表于 2019-12-25

一、题意

题目链接: 501.Find Mode in Binary Search Tree

1.Description

【输入】
给定一棵特殊的二叉树。
这棵二叉树允许：对于任一节点，其左、右儿子的值允许与其相等。而不必严格的小于、大于。
【输出】
找出这棵二叉树中的众数（Mode），即出现次数最多的数字。
众数可能不止一个，以一维数组的形式返回。

2.Example

Input: [1,null,2,2]
Output: [2]

3.Corner Case

  1. 当树为NULL时，返回空数组。

二、题解

Solution-1：Iterative Solution(Inorder)

1.虽然是特殊的BST，但其inorder序列仍然是一个非递减序列。

2.preValue：前一个数字
maxCnt：众数出现的数字
curCnt：当前数字出现的次数

3.检查当前数字和前一个数字的值：
如果当前数字等于前一个数字，则：curCnt++
如果当前数字不等于前一个数字，则：curCnt = 1; preValue = temp->val;

4.检查当前次数和之前出现过的最高次数：
如果相等，则将当前数字加入ans
如果大于，则将ans清空，将当前数字加入ans，更新maxCnt。

时间复杂度：O(n)
空间复杂度：O(n)
