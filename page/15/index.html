【链表专项】160.Intersection of Two Linked Lists

发表于 2019-12-30

一、题意

题目链接: 160.Intersection of Two Linked Lists

1.Description

【输入】
给定两个单链表。
【输出】
判断这两个链表是否在某个结点相交。如果相交，找出这个交点。否则的话，返回NULL即可。

2.Example

listA = [0,9,1,2,4]
listB = [3,2,4]
返回结点2。

3.Corner Case

  1. 若一个链表为NULL，另一个不为NULL；或者两个都为NULL，则直接返回NULL。
  2. 函数结束后，原有的链式结构必须和之前完全一样。即不能修改这两个链表。
  3. 题目保证链表无环。
  4. 要求O(n)的time complexity
  5. 要求O(1)的space complexity

二、题解

Solution-1：Two Pointers

1.假设两个链表相交的话，相交前，各自的距离分别为a、b。相交后共同的距离为c。
则p、q分别从两个链表头headA、headB出发：若p当前为NULL，则将其转移到headB。若q当前为NULL，则将其转移到headA。
如果相交的话，p、q一定会在交点相遇。此时直接返回该点即可。

2.原理：
p走的距离：a + c + b
q走的距离：b + c + a
此时p、q一定相遇。

3.若两个链表本来就不相交的话，各自的距离分别为a、b。
p走的距离：a + b
q走的距离：b + a
此时p、q一定相遇。而此时p、q正好均为NULL。将其返回即可。

时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】019.Remove Nth Node From End of List

发表于 2019-12-30

一、题意

题目链接: 019.Remove Nth Node From End of List

1.Description

【输入】
给定一个单链表，头为head。
给定一个正整数n：[0, num]。其中num是链表中结点的数目。
【输出】
将上述单链表中从后面数第n个结点删除掉。
注意：编号不管从前还是从后，都是从1开始。

2.Example

Given linked list: 1->2->3->4->5, and n = 2.
After removing the second node from the end, the linked list becomes 1->2->3->5.

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. n为0时，无需处理。
  3. one pass处理。即：仅仅遍历链表一次。

二、题解

Solution-1：Iterative Solution

1.常规思路：先获取到链表中有多少结点，然后推算出来从前往后是要删除哪一个节点。
再经过一次遍历，删除对应的结点。

2.本题要求one pass处理，即不允许遍历链表两次。可以使用快慢指针法：
slow指针初始化为head
fast指针初始化为head，fast先向后走n步（1 <= n <= num）。此时检查fast是否为NULL。
a.如果NULL，说明n == num，要删除链表中第一个元素。
b.如果不为NULL的话，说明n < num，要删除的结点非head。

3.此时slow，fast同时出发，它们之间相距n个结点。
它们俩同时往后移动，直到fast遍历到链表中的最后一个结点，停止。此时slow->next即要删除的结点。

时间复杂度：O(n)
空间复杂度：O(1)
