【BST专项】235.Lowest Common Ancestor of a Binary Search Tree

发表于 2019-12-26

一、题意

题目链接: 235.Lowest Common Ancestor of a Binary Search Tree

1.Description

【输入】
给定一棵BST。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [6,2,8,0,4,7,9,null,null,3,5], p = 2, q = 8
Output: 6
Explanation: The LCA of nodes 2 and 8 is 6.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

Solution-1：Recursive Solution

1.思路：根据BST的性质：对于任意node来说，左子树中的所有节点都小于node的值；右子树中的所有节点都大于node的值；

2.具体方法：
从root节点开始，若p和q的val均小于root的val，则结果一定在root的左子树中
若p和q的val均大于root的val，则结果一定在root的右子树中
若一个大于root的val，一个小于，则pq的LCA一定为root

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276781092/)

Solution-2：Iterative Solution

1.减少递归栈的space complexity

2.具体方法：
每次修改root节点，将其赋值为其左儿子或者其右儿子。
缺点：修改了原始数据，使用前需确认评估。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/276783571/)

【BST专项】230.Kth Smallest Element in a BST

发表于 2019-12-26

一、题意

题目链接: 230.Kth Smallest Element in a BST

1.Description

【输入】
给定一棵BST。
给定一个整数k。（题目保证k合法，即 1 <= k <= n，n为BST的节点数）
【输出】
求出这棵BST中第k小的元素的值，以int形式返回。

2.Example

Input: [3,1,4,null,2], k = 1
Output: 1

3.Corner Case

  1. 树为NULL时，直接return 0。

二、题解

Solution-1：Iterative Solution(DFS-Inorder)

1.对BST进行中序遍历，并累加cnt当前节点是遍历的第几个节点。若cnt == k，则该节点就是答案。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276741636/)

【BST专项】501.Find Mode in Binary Search Tree

发表于 2019-12-25

一、题意

题目链接: 501.Find Mode in Binary Search Tree

1.Description

【输入】
给定一棵特殊的二叉树。
这棵二叉树允许：对于任一节点，其左、右儿子的值允许与其相等。而不必严格的小于、大于。
【输出】
找出这棵二叉树中的众数（Mode），即出现次数最多的数字。
众数可能不止一个，以一维数组的形式返回。

2.Example

Input: [1,null,2,2]
Output: [2]

3.Corner Case

  1. 当树为NULL时，返回空数组。

二、题解

Solution-1：Iterative Solution(Inorder)

1.虽然是特殊的BST，但其inorder序列仍然是一个非递减序列。

2.preValue：前一个数字
maxCnt：众数出现的数字
curCnt：当前数字出现的次数

3.检查当前数字和前一个数字的值：
如果当前数字等于前一个数字，则：curCnt++
如果当前数字不等于前一个数字，则：curCnt = 1; preValue = temp->val;

4.检查当前次数和之前出现过的最高次数：
如果相等，则将当前数字加入ans
如果大于，则将ans清空，将当前数字加入ans，更新maxCnt。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/288451703/)

【BST专项】098.Validate Binary Search Tree

发表于 2019-12-25

一、题意

题目链接: 098.Validate Binary Search Tree

1.Description

【输入】
给定一棵树。
【输出】
判断其是否为一棵BST(Binary Search Tree)(Binary Sort Tree)
【背景】
BST（二叉搜索树）（二叉排序树）
定义：
a.若左子树非空，左子树中所有节点的值都比根节点的值小。
b.若右子树非空，右子树中所有节点的值都比根节点的值大。
c.左子树、右子树都是BST。
特点：
a.树中任一节点的值是唯一的。
b.中序遍历序列是一个严格升序序列。

2.Example

Input: [5,1,4,null,null,3,6]
Output: false
Explanation: 节点4在根节点5的右子树，矛盾。

3.Corner Case

  1. 树为NULL时，true。
  2. 树为单节点时，true。

二、题解

Solution-1：Recursive Solution(Preorder)

1.遍历顺序为preorder。

2.从root开始，依次dfs遍历到每一个节点
对于当前节点node来说：
我的父亲调用我的时候，把他期望我所在的取值范围告诉我：(left, right)
首先：我判断自己是否在这个范围内，若不在，说明我的val不满足BST的性质
其次：我需要判断左子树：我期望其在以下取值范围内：(left, node->val)
右子树：我期望其在以下取值范围内：(node->val, right)
如果以上都满足，则开开心心的返回true。有哪一环不满足，都返回false。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276727177/)

Solution-2：Iterative Solution(Inorder)

1.遍历顺序为inorder。

2.每次比较当前值和前一个值的大小。只有当前值严格大于前一个值，才符合BST的性质。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276738088/)

【二叉树专项】129.Sum Root to Leaf Numbers

发表于 2019-12-25

一、题意

题目链接: 129.Sum Root to Leaf Numbers

1.Description

【输入】
给定一棵二叉树，所有节点的值均在[0, 9]中间。
每条从root到leaf的路径都可以代表一个整数。
【输出】
求出所有从root到leaf的路径，代表的整数的和。

2.Example

Input: [1,2,3]
Output: 25

3.Corner Case

  1. 树为NULL时，sum为0
  2. 树为单节点时，sum为node->val

二、题解

Solution-1：Recursive Solution(DFS)

1.遍历节点顺序为DFS(Postorder)

2.对于任一节点：
判断自身：如果自身为NULL，直接返回0；如果自身为叶子，则值 = cur->val + curSum * 10，上报给父亲。
询问儿子：将自身的值(cur->val + curSum * 10)，传递给儿子，要求它们计算。
计算自身：左儿子的值 + 右儿子的值
上报父亲：将自身的值上报给父亲

时间复杂度：O(n)
空间复杂度：O(n)
