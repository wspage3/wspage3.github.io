【BST专项】108.Convert Sorted Array to Binary Search Tree

发表于 2019-12-25 更新于 2019-12-27

一、题意

题目链接: 108.Convert Sorted Array to Binary Search Tree

1.Description

【输入】
给定一个长度为n的数组，数组是升序排列的。
【输出】
将这个数组转化为一个高度平衡的（height-balanced）二叉搜索树（BST）。

2.Example

Input: [-10,-3,0,5,9]
Output: [0,-3,9,-10,NULL,5,NULL] 答案不唯一。

3.Corner Case

  1. n = 0时，树为NULL。
  2. n = 1时，树为单节点。

二、题解

Solution-1：Recursive Solution

1.给定一个长度为n的数组nums，要将其转化为高度平衡的BST。
a.取数组中间元素作为root；
b.递归将其左边所有元素构成一棵较小的高度平衡的BST。作为root的左子树。
c.递归将其右边所有元素构成一棵较小的高度平衡的BST。作为root的右子树。

2.结束条件：若数组长度为1，则将其作为一个单节点。

3.不管n是奇数还是偶数，对于任一节点，其左右子树的节点数不会大于1。保证了height-balanced。

时间复杂度：O(n)
空间复杂度：O(n)

4.Code(https://leetcode.com/submissions/detail/277835456/)

Solution-2：Recursive Solution with Optimization

1.优化：递归调用的时候，避免：构建子数组，将子数组作为参数传递。
而是直接传递索引即可。

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/288910868/)

【二叉树专项】437.Path Sum III

发表于 2019-12-24 更新于 2019-12-25

一、题意

题目链接: 437.Path Sum III

1.Description

【输入】
给定一棵二叉树。
给定一个整数target。
【输出】
判断是否存在一条路径，使得路径上所有节点的sum为target。
且路径只能是向下的，即从父亲到儿子。
如果存在，返回可能的路径数。
如果不存在，返回0。

2.Example

Input:
8
[10,5,-3,3,2,null,11,3,-2,null,1]
Output:
Return 3

  1. 5 -> 3
  2. 5 -> 2 -> 1
  3. -3 -> 11

3.Corner Case

  1. 节点数：n <= 1000
  2. 节点值value：[-1000000, 1000000]
  3. 树为NULL时，返回空二维数组。
  4. 树仅包含一个node时，查看node->val 是否等于target。

二、题解

Solution-1：Recursive solution(Dual DFS)

1.遍历顺序为dfs的顺序

2.求一颗非空树root的答案，也就是树中任意路径和为target的方案数，可以分解为以下三种情况：
a.路径起点是root节点
b.路径起点不是root节点，也就是说root节点和该问题的答案没任何关系，它的存在不影响问题的解。
此时将问题转化为：在root->left这个子树中找答案。在root->right这个子树中找答案。则问题答案就是这两个子答案的和。

时间复杂度：O(n ^ 2) //最坏单链表的情况 最好平衡树的情况O(n * log n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/278655172/)

Solution-2：PrefixSum + Cache

1.遍历顺序依旧是dfs。

2.采用prefixSum的方法，避免进行过多的重复计算。
遍历到某个节点时，将从root到该节点这条路径的和累加，缓存起来，在后续过程，可以利用这些有用信息。

3.具体过程如下：
a.访问到每一个节点的时候，cur += node->val；
b.判断(cur - sum)有没有在字典中出现，有的话，答案累加其出现的频率（次数）
【原理1】：当遍历到当前节点node时，假设cur为x，也就是sum(root, node) = x。若字典中存在(x - sum)，说明在某个节点node1，sum(root, node1) = (x - sum)。
那么也就可以推断出来：sum(node1->next, node) = sum。也就是答案要求解的路径。
【原理2】：字典中存在的记录，一定是从root到node路径上所有节点的前缀和。也就是说保证了某个节点到当前节点node的【可通性】。因此，我们在遍历完每个节点后，要将其在字典中的“影响”消除。
c.将当前cur缓存到字典中，其出现频率加一。
至此，当前节点node“自身”考虑完成。

4.接下来递归考虑node的左儿子和右儿子。
当他们都考虑完成的时候，节点node才真正考虑完成。此时将cur在字典中出现的频率减一。
同时返回给父亲节点（回溯）

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/278714068/)

【二叉树专项】113.Path Sum II

发表于 2019-12-24 更新于 2019-12-26

一、题意

题目链接: 113.Path Sum II

1.Description

【输入】
给定一棵二叉树。
给定一个整数target。
【输出】
判断是否有一条从root到leaf的路径，这条路径上所有节点的sum为target。
如果存在，则返回一个二维数组，代表有n条路径满足题意，每一条路径是一个一维数组。
如果不存在，返回空的二维数组。

2.Example

Input: 22
[5,4,8,11,null,13,4,7,2,null,null,5,1]
Output:
[
[5,4,11,2],
[5,8,4,5]
]

3.Corner Case

  1. 树为NULL时，返回空二维数组。
  2. 树仅包含一个node时，查看node->val 是否等于target。

二、题解

Solution-1：Recursive solution(DFS)(Backtracking)

1.遍历顺序为dfs的顺序

2.对于任一节点来说：
父亲调用自身的时候：
a.传递一个值target下来。
b.传递一个路径下来，存在一维数组sub中。

3.叶节点需要做：
首先，将自己放在路径中。
其次，判断自身（如果自身value和target相等，那么将这条路径作为一个合法的子答案加入res；如果不相等的话，不加入ans）
然后，删除自身。因为本节点已经被考虑完成了，也不可能有后继节点了。所以从sub中删除本节点。
最后，上报父亲。return即可。

4.非叶节点需要做：
首先，将自己放在路径中。
其次，判断自身（自身一定不满足，因为不是叶子）
然后，判断非空的儿子。将target - node->val传递下去。
接着，删除自身。因为本节点自身不满足，而其非空儿子都考虑完成了。所以从sub中删除本节点。
最后，上报父亲。return即可。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276102013/)

Solution-2：Iterative solution using Stack(DFS)

1.此处代码较丑陋，主要为了使用stack模拟递归。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/288669284/)

【二叉树专项】112.Path Sum

发表于 2019-12-24

一、题意

题目链接: 112.Path Sum

1.Description

【输入】
给定一棵二叉树。
给定一个整数target。
【输出】
判断是否有一条从root到leaf的路径，这条路径上所有节点的sum为target。
存在返回true，不存在返回false。

2.Example

Input: 22
[5,4,8,11,null,13,4,7,2,null,null,null,1]
Output: true(5->4->11->2)

3.Corner Case

  1. 树为NULL时，返回false。
  2. 树仅包含一个node时，return node->val == target ? true : false。

二、题解

Solution-1：Recursive solution(DFS)

1.遍历顺序为dfs的顺序

2.对于任一节点来说：
父亲调用自身的时候，传递一个值target下来。
判断自身（如果自身是叶子节点的话，直接判断自身value是否和target相等）
判断两儿子（递归调用，将target - node->val传递给左右儿子）
上报给父亲（返回结果）

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276069188/)

Solution-2：Iterative solution using Stack(DFS-Preorder Traversal)

1.遍历顺序为dfs的顺序

2.开两个stack，一个用来遍历node，一个用来存相同位置上的node对应的target。
使用iterative版本preorder的顺序遍历每一个node。
每次从两个栈顶pop：考察一个node，考察一个value。
如果该node是leaf的话，对其进行检查：如果刚刚抛出的value为0，则说明存在一条满足题意的路径。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276075062/)

【二叉树专项】226.Invert Binary Tree

发表于 2019-12-24

一、题意

题目链接: 226.Invert Binary Tree

1.Description

【输入】
给定一棵二叉树。
【输出】
将其翻转。
也就是求出与它互为镜像的二叉树。

2.Example

Input: [4, 2, 7, 1, 3, 6, 9]
Output: [4, 7, 2, 9, 6, 3, 1]

3.Corner Case

  1. 树为NULL时，返回NULL。
  2. 树仅包含一个node时，返回这个节点。

二、题解

Solution-1：Recursive solution(DFS)

1.遍历顺序为dfs的顺序

2.对于任一节点来说：
通知两儿子（递归调用，使得两儿子分别进行invert操作）
自己计算（将自己的左儿子更新为：右儿子invert后返回的值；将右儿子更新为：左儿子invert后返回的值）
上报给父亲（返回自己）

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275865590/)

Solution-2：Iterative solution using Queue(BFS)

1.遍历顺序为bfs的顺序

2.每次遍历到一个节点node时，将其左右儿子交换。然后再将其放入队列。
这样的话，每个节点的左右儿子都被交换过。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275868273/)
