【DP专项】518.Coin Change 2

发表于 2020-01-03

一、题意

题目链接: 518.Coin Change 2

1.Description

【输入】
给定一个长度为n的整数数组，代表着有这些面值的硬币。
给定一个整数target。
【输出】
求出有多少种不同的组合方式，使得组合起来的sum等于target。
每个硬币可以使用无限次。
注意：硬币的顺序不重要。就像去超市买东西一样，不会说把硬币一个一个拿出来，而是一把拿出来。

2.Example

Input: amount = 5, coins = [1, 2, 5]
Output: 4
Explanation: there are four ways to make up the amount:
5=5
5=2+2+1
5=2+1+1+1
5=1+1+1+1+1

3.Corner Case

  1. 0 <= amount <= 5000
  2. 1 <= coins[i] <= 5000
  3. n < 500
  4. the answer is guaranteed to fit into signed 32-bit integer

二、题解

思路

  1. 当amount为0时，不管n是否为0，答案为1。即：有且仅有一种组合方式使得amount为0，那就是什么都不选。
  2. 当amount不为0，n为0时，答案为0。即：不存在一种组合方式，使得sum等于amount。
  3. 考虑amount大于0，n大于0的场景。

Solution-1：Dynamic Programming(Iterative)

  1. 参考(https://www.youtube.com/watch?v=DJ4a7cmjZY0)

  2. 回忆【377.Combination Sum IV】中，也是给定一组数，和一个target，求多少种不同的组合方式，使得sum为target。
    但是不同元素的顺序是重要的。即122和221是两种不同的组合方式，需要都计数。

  3. 回忆【322.Coin Change】，求得最小使用了几个硬币，使得sum为target。

  4. 状态定义：dp[i][j]代表：仅仅使用了coins数组的前i种硬币，使得和为j的方案数。

  5. 转移方程：dp[i][j] = dp[i - 1][j] + dp[i][j - coins[i - 1]]。
    解释：从前到后考虑每一类型的硬币，求前i种硬币使得sum为j的组合方式：
    先求前(i - 1)种硬币使得sum为j的组合方式；即：dp[i - 1][j]。即：完全不用当前硬币组成j有多少种组合。
    再求前i种硬币使得sum为(j - coins[i - 1])的组合方式，即：dp[i][j - coins[i - 1]]。即：使用至少一个当前硬币组成组成j有多少组合。

  6. 边界条件：
    dp[i][0] = 1；0 <= i <= n
    dp[0][j] = 0；1 <= j <= target

  7. 求解顺序：
    【Backward】
    i从1开始，到n结束。
    j从1开始，到target结束。
    依次考虑每一种类型的硬币，求解出来的子问题保存起来。为后面求解更大一点的问题服务。

  8. 空间复杂度压缩。

  9. Code(https://leetcode.com/submissions/detail/290726614/)
    时间复杂度：O(n * amount)
    空间复杂度：O(n * amount)

  10. Code(https://leetcode.com/submissions/detail/290745421/)
    时间复杂度：O(n * amount)
    空间复杂度：O(amount)

【DP专项】322.Coin Change

发表于 2020-01-03

一、题意

题目链接: 322.Coin Change

1.Description

【输入】
给定一个长度为n的整数数组。代表着你拥有这些面值的硬币。
给定一个正整数target。
【输出】
求出你最少需要多少个硬币，使得它们的钱的sum等于target。
如果拼不出来，则返回-1。

2.Example

Input: coins = [1, 2, 5], amount = 11
Output: 3
Explanation: 11 = 5 + 5 + 1

3.Corner Case

  1. 每种硬币可以使用无限次。
  2. 每种硬币的面值都大于0。那么当target小于0时，ans = -1。

二、题解

Solution-1：Dynamic Programming(Iterative)

  1. 思考：
    Q: What is important so far???
    A: The sum、minimum coins

  2. 状态定义：dp[i]：sum为i，所需要的硬币的最小个数。本题答案即dp[target]。

  3. 转移方程：dp[i] = min(dp[i], 1 + dp[i - x])。其中(i - x)大于等于0；x是nums中任一元素。

  4. 边界条件：dp[0] = 0；在计算dp[i]前，将其初始化为INT_MAX；

  5. 求解方向：
    【Backward DP】
    i从1开始，到target结束。通过dp[i - x]来计算求得dp[i]。

【Forward DP】
i从0开始，到target - 1结束。使用dp[i]来更新dp[i + x]。

  1. 空间优化：不能降维。

  2. Code(https://leetcode.com/submissions/detail/290651014/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)

  3. Code(https://leetcode.com/submissions/detail/290653511/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)

【DP专项】377.Combination Sum IV

发表于 2020-01-02

一、题意

题目链接: 377.Combination Sum IV

1.Description

【输入】
给定一个长度为n的整数数组。数组元素都是正数。
给定一个正整数target。
【输出】
求出有多少种不同的组合方式，使得组合的sum等于target。
每个数字可以使用任意次。

2.Example

nums = [1, 2, 3]
target = 4

The possible combination ways are:
(1, 1, 1, 1)
(1, 1, 2)
(1, 2, 1)
(1, 3)
(2, 1, 1)
(2, 2)
(3, 1)

Note that different sequences are counted as different combinations.
Therefore the output is 7.

3.Corner Case

  1. 数组元素保证正数
  2. target保证正数
  3. 当n为0时，不能使得sum等于正数的target，所以答案为0

4.Follow Up

  1. 如果数组元素中有负数呢？

二、题解

Solution-1：Dynamic Programming(Iterative)

  1. 思考：
    Q: What is important so far???
    A: The sum

  2. 状态定义：dp[i]：代表有多少种不同的方式组成一个序列，使得序列和为i。

  3. 转移方程：dp[i] += dp[i - x]。其中(i - x)必须大于等于0；x是nums中出现的所有元素。
    解释：要想使得sum为i，那么一定是由(i - x)转移过来。
    举个例子：nums = [1, 2, 3]，target = 20。
    当计算到dp[10]的时候：10 = 9 + 1 = 8 + 2 = 7 + 3。
    所以：dp[10] = dp[9] + dp[8] + dp[7]

  4. 边界条件：dp[0] = 1。

  5. 求解方向：
    【Backward DP】
    i从1开始，到target结束。
    对于每个i，遍历nums所有的元素。如果(i - x)大于等于0，则dp[i]累加dp[i - x]。

【Forward DP】
已知dp[0] = 1。
i从0开始，到target - 1结束。因为最多只可能target - 1转移到target。
对于每个i，遍历nums所有的元素。如果(i + x)小于等于target，则dp[i + x]累加dp[i]。

  1. 空间优化：本题目无法进行空间降维。因为计算最后一项dp[target]的时候，可能使用到很多项之前的dp[i]。

  2. 注意：当用例为：[3,33,333]，10000时，可以看出dp[10000]其实为0，因为不管这么组合，都不可能使得sum为10000。
    此时在从1计算到10000的时候，存在某些i，使得dp[i]非常大，使用long long也不足以表示其方案数。
    此时，可以换成无符号的int来存放值。这样，当overflow的时候，unsigned int会自动处理溢出。而不会报Runtime Error。
    For example, unsigned int x = 4294967295 is ok but after do x += 1 operation, x will be 0 without any runtime error.

  3. Code(https://leetcode.com/submissions/detail/290501613/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)

  4. Code(https://leetcode.com/submissions/detail/290506960/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)

【DP专项】120.Triangle

发表于 2020-01-02

一、题意

题目链接: 120.Triangle

1.Description

【输入】
给定一个二维数组。代表着一个三角形。
【输出】
求出一条从顶点到最底层任意一点的路径，使得这条路径上的所有结点的和最小。

2.Example

[
[2],
[3,4],
[6,5,7],
[4,1,8,3]
]
The minimum path sum from top to bottom is 11 (i.e., 2 + 3 + 5 + 1 = 11).

3.Corner Case

  1. m为1，n为1时，答案就是triangle[0][0]
  2. m为2，n为2时，答案就是triangle[0][0] + min(triangle[1][0], triangle[1][1])
  3. m与n相等

二、题解

Solution-1：Dynamic Programming(Iterative)

  1. 状态定义：dp[i][j]：从最低端，到达位置(i, j)的所有路径中，最短的那条路径对应的和。答案就是dp[0][0]。

  2. 转移方程：dp[i][j] = min(dp[i + 1][j], dp[i + 1][j + 1]) + triangle[i][j]

  3. 边界条件：当i == n - 1时，dp[i][j] = triangle[i][j]

  4. 求解方向：从倒数第二层开始计算，即i从n - 2开始，到0结束。
    这样的【Bottom-Up】的求解方向，使得遍历完，结果就是dp[0][0]了。
    如果采用【Top-Down】的求解方向，即i从1开始，到n - 1结束。这样的话，需要考虑i - 1，以及j - 1是否存在的问题，即多了if语句判断。
    而且，最后需要找到第n - 1层的最小值。

  5. 空间优化：不另开dp数组，复用triangle数组即可。

  6. Code(https://leetcode.com/submissions/detail/261926210/)
    时间复杂度：O(m * n)
    空间复杂度：O(1)

【DP专项】213.House Robber II

发表于 2020-01-02

一、题意

题目链接: 213.House Robber II

1.Description

【输入】
给定一个长度为n的整数数组。数组元素都是非负的。代表着n户人家的积蓄。
【输出】
计算出你能抢劫到最多多少钱。
不允许抢劫两个相邻的人家，会导致自动报警。
同时，这些人家绕成了一个圈。这意味着：第一户人家和最后一户人家是相邻的。

2.Example

Input: [2,3,2]
Output: 3
Explanation: You cannot rob house 1 (money = 2) and then rob house 3 (money = 2),
because they are adjacent houses.

3.Corner Case

  1. n为0时，答案为0。
  2. n为1时，答案为nums[0]。
  3. n为2时，答案为max(nums[0], nums[1])。

二、题解

Solution-1：Dynamic Programming

  1. 状态定义：dp[i]：抢劫完前i户([0, i])人家，最多能获得的钱。

  2. 转移方程：dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])

  3. 边界条件：dp[0] = nums[0], dp[1] = max(nums[0], nums[1])

  4. 求解顺序：
    i从0开始，到n - 2结束，结果记为ans1。
    i从1开始，到n - 1结束，结果记为ans2。
    答案即：max(ans1, ans2)。

  5. Code(https://leetcode.com/submissions/detail/272474951/)
    时间复杂度：O(n)
    空间复杂度：O(1)
