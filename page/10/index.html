【DP专项】509.Fibonacci Number

发表于 2020-01-02

一、题意

题目链接: 509.Fibonacci Number

1.Description

【输入】
给定一个非负整数n。
【输出】
求第n项的Fibonacci值。
F(0) = 0, F(1) = 1
F(N) = F(N - 1) + F(N - 2), for N > 1.

2.Example

Input: 4
Output: 3
Explanation: F(4) = F(3) + F(2) = 2 + 1 = 3.

3.Corner Case

  1. n为0时，答案为0
  2. n为1时，答案为1
  3. 0 ≤ n ≤ 30

Solution-1：Dynamic Programming(Iterative)

1.回忆【070.Climbing Stairs】，本质上就是求Fibonacci数列的第n项。
只不过base case需要特定考虑。

时间复杂度：O(n)
空间复杂度：O(1)

【DP专项】070.Climbing Stairs

发表于 2020-01-01 更新于 2020-01-02

一、题意

题目链接: 070.Climbing Stairs

1.Description

【输入】
给定一个正整数n，代表有n个台阶。
【输出】
假设你每次只能走一步或者两步。
求你达到楼顶，有多少种不同的方式？

2.Example

Input: 3
Output: 3
Explanation: There are three ways to climb to the top.

  1. 1 step + 1 step + 1 step
  2. 1 step + 2 steps
  3. 2 steps + 1 step

3.Corner Case

  1. n为1时，答案为1
  2. n为2时，答案为2
  3. 题目保证n是正整数

二、题解

分析

  1. 状态定义：dp[i]：有多少种不同的方法到达第i个台阶。
  2. 转移方程：dp[i] = dp[i - 2] + dp[i - 1]；答案就是dp[n - 1]；
  3. 边界条件：dp[0] = 1, dp[1] = 2
  4. 计算顺序：从2开始，到n - 1结束。

Solution-1：Dynamic Programming(Iterative)

1.dp[i] = dp[i - 2] + dp[i - 1]

2.Code(https://leetcode.com/submissions/detail/261623472/)
时间复杂度：O(n)
空间复杂度：O(n)

Solution-2：Dynamic Programming(Iterative) + Space Optimization

1.dp[i] = dp[i - 2] + dp[i - 1]

2.Code(https://leetcode.com/submissions/detail/261625030/)
时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】143.Reorder List

发表于 2019-12-30

一、题意

题目链接: 143.Reorder List

1.Description

【输入】
给定一个单链表：L0→L1→…→Ln-1→Ln。
【输出】
将其转换为：L0→Ln→L1→Ln-1→L2→Ln-2→…
返回值void。

2.Example

Given 1->2->3->4->5, reorder it to 1->5->2->4->3.

3.Corner Case

  1. 如果链表为NULL，则无需处理。
  2. 如果链表为节点数小于等于2，无需处理。

二、题解

Solution-1：Fast and Slow Pointers

1.回忆【234.Palindrome Linked List】中判断回文链表：找到中间点，顺带着reverse了前半部分。然后与后半部分的结点逐一比较。

2.本题中，我们先利用快慢指针找到链表中间点。然后reverse后半部分链表。最后merge两部分。

3.举个例子1：
链表为：[1->2->3->4->NULL]
经过快慢指针，fast：NULL。slow：3。cur：4。
以cur为首结点，进行reverse：[1->2->3->NULL] [4->NULL]
此时：pre指向4。prepre指向1。
进行merge：[1->4->2->3->NULL]

4.举个例子2：
链表为：[1->2->3->4->5->NULL]
经过快慢指针，fast：5。slow：3。cur：4。
以cur为首结点，进行reverse：[1->2->3->NULL] [5->4->NULL]
此时：pre指向5。prepre指向1。
进行遍历，交替拼接：[1->5->2->4->3->NULL]

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/289773356/)

【链表专项】160.Intersection of Two Linked Lists

发表于 2019-12-30

一、题意

题目链接: 160.Intersection of Two Linked Lists

1.Description

【输入】
给定两个单链表。
【输出】
判断这两个链表是否在某个结点相交。如果相交，找出这个交点。否则的话，返回NULL即可。

2.Example

listA = [0,9,1,2,4]
listB = [3,2,4]
返回结点2。

3.Corner Case

  1. 若一个链表为NULL，另一个不为NULL；或者两个都为NULL，则直接返回NULL。
  2. 函数结束后，原有的链式结构必须和之前完全一样。即不能修改这两个链表。
  3. 题目保证链表无环。
  4. 要求O(n)的time complexity
  5. 要求O(1)的space complexity

二、题解

Solution-1：Two Pointers

1.假设两个链表相交的话，相交前，各自的距离分别为a、b。相交后共同的距离为c。
则p、q分别从两个链表头headA、headB出发：若p当前为NULL，则将其转移到headB。若q当前为NULL，则将其转移到headA。
如果相交的话，p、q一定会在交点相遇。此时直接返回该点即可。

2.原理：
p走的距离：a + c + b
q走的距离：b + c + a
此时p、q一定相遇。

3.若两个链表本来就不相交的话，各自的距离分别为a、b。
p走的距离：a + b
q走的距离：b + a
此时p、q一定相遇。而此时p、q正好均为NULL。将其返回即可。

时间复杂度：O(n)
空间复杂度：O(1)
