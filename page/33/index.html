【二叉树专项】106.Construct Binary Tree from Inorder and Postorder Traversal

发表于 2019-12-27 更新于 2019-12-29

一、题意

题目链接: 106.Construct Binary Tree from Inorder and Postorder Traversal

1.Description

【输入】
给定两个二维数组，分别代表一棵二叉树的：
后序遍历序列
中序遍历序列
【输出】
求出这棵二叉树。

2.Example

Input:
inorder = [9,3,15,20,7]
postorder = [9,15,7,20,3]

Output: [3,9,20,NULL,NULL,15,7]

3.Corner Case

  1. 目标数组为空时，树为NULL。
  2. 目标数组长度为1时，树为单节点。

二、题解

Solution-1：Recursive Solution(Postorder)

1.根据以下特性：
postorder序列的最后一个数字一定是：root的值
inorder序列：root的值左边都是root的左子树，右边都是root的右子树。

2.对于任意任意两个序列：build(postStart, postEnd, inStart, inEnd)
a.根节点root一定是postorder[postEnd]
b.在inorder中找到root的位置，记为pos。则左子树的节点数目：pos - inStart
c.递归构建左子树：build(postStart, postStart + (pos - inStart) - 1, inStart, pos - 1)
d.递归构建右子树：build(postStart + (pos - inStart), postEnd - 1, pos + 1, inEnd)
e.将c.d两步的返回值赋值为root的左子树和右子树，return给上一层。

时间复杂度：O(n)
空间复杂度：O(n)

【二叉树专项】236.Lowest Common Ancestor of a Binary Tree

发表于 2019-12-26

一、题意

题目链接: 236.Lowest Common Ancestor of a Binary Tree

1.Description

【输入】
给定一棵二叉树。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4
Output: 5
Explanation: The LCA of nodes 5 and 4 is 5, since a node can be a descendant of itself according to the LCA definition.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

思路

本题不同于上一题，上一题中输入的树为BST，可以通过比较节点的数值，每遍历一个节点，将问题规模缩小到原来的一半。

Solution-1：Recursive Solution

1.如果root为NULL，则return NULL

2.如果p,q中一者为root，则它俩的LCA一定为root

3.否则的话，说明：p和q一定是root的某一个子节点，分别在root的左子树和右子树中查找p,q的LCA
a.若在某个子树中找不到p和q，则代表p和q同时在本子树的兄弟子树中，此时返回NULL
b.若在某个子树中找到了p（或者q），则函数立马返回p（或者q）。
相当于说：某个后代M发现了两个目标（p或q）中的其中一个（例如p），立马报告自己的父亲A。
由于A还有一个儿子N，当M和N都给A汇报了结果后，A再进行决策。
a.若此时N也发现了目标中的另外一个，则A就是p和q的LCA。
b.若此时N没有发现目标，则A上报他自己的父亲B：p是属于以A为根的子树中的某个节点。
c.B收到这个信息，同时再看B的另外一个儿子的上报信息，决策并往上上报信息，直至root节点。
d.假设B在root的左子树中，root一看，发现p在其左子树中。同时root收到了其右子树的信息。
假设右子树中发现了q，此时root就可以说自己是p和q的LCA
假设右子树中发现了p（不可能的情况，p已在root的左子树中被发现了）
假设右子树中没有发现p,q的任何一个，即返回值为NULL。此时root一想，p已被发现，而q没有，那么p一定就是二者的LCA，则p为答案。
因为我们在发现了二者中的任意一个的时候，都立马返回，上报给父亲。而访问的顺序是dfs，说明未被发现的q一定为p的某个后代，故此p就是答案。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277036953/)

Solution-2：Iterative Solution

1.思路2：
找出从root到p的路径，存放在vectorP中：root, a, b, c, …, p
找出从root到q的路径，存放在vectorP中：root, a, b, c, …, q
从前往后对比两个vector，最后一个匹配的节点，即是答案。

2.在找从root到p、q的路径时，用dfs(preorder)的顺序：
遍历到某个节点，将其加入路径中。
如果这个节点为p或者q其中一个，则一条路径寻找完成。直到两条路径都找到，此时结束遍历。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277045028/)

【BST专项】235.Lowest Common Ancestor of a Binary Search Tree

发表于 2019-12-26

一、题意

题目链接: 235.Lowest Common Ancestor of a Binary Search Tree

1.Description

【输入】
给定一棵BST。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [6,2,8,0,4,7,9,null,null,3,5], p = 2, q = 8
Output: 6
Explanation: The LCA of nodes 2 and 8 is 6.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

Solution-1：Recursive Solution

1.思路：根据BST的性质：对于任意node来说，左子树中的所有节点都小于node的值；右子树中的所有节点都大于node的值；

2.具体方法：
从root节点开始，若p和q的val均小于root的val，则结果一定在root的左子树中
若p和q的val均大于root的val，则结果一定在root的右子树中
若一个大于root的val，一个小于，则pq的LCA一定为root

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276781092/)

Solution-2：Iterative Solution

1.减少递归栈的space complexity

2.具体方法：
每次修改root节点，将其赋值为其左儿子或者其右儿子。
缺点：修改了原始数据，使用前需确认评估。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/276783571/)

【二叉树专项】437.Path Sum III

发表于 2019-12-24 更新于 2019-12-25

一、题意

题目链接: 437.Path Sum III

1.Description

【输入】
给定一棵二叉树。
给定一个整数target。
【输出】
判断是否存在一条路径，使得路径上所有节点的sum为target。
且路径只能是向下的，即从父亲到儿子。
如果存在，返回可能的路径数。
如果不存在，返回0。

2.Example

Input:
8
[10,5,-3,3,2,null,11,3,-2,null,1]
Output:
Return 3

  1. 5 -> 3
  2. 5 -> 2 -> 1
  3. -3 -> 11

3.Corner Case

  1. 节点数：n <= 1000
  2. 节点值value：[-1000000, 1000000]
  3. 树为NULL时，返回空二维数组。
  4. 树仅包含一个node时，查看node->val 是否等于target。

二、题解

Solution-1：Recursive solution(Dual DFS)

1.遍历顺序为dfs的顺序

2.求一颗非空树root的答案，也就是树中任意路径和为target的方案数，可以分解为以下三种情况：
a.路径起点是root节点
b.路径起点不是root节点，也就是说root节点和该问题的答案没任何关系，它的存在不影响问题的解。
此时将问题转化为：在root->left这个子树中找答案。在root->right这个子树中找答案。则问题答案就是这两个子答案的和。

时间复杂度：O(n ^ 2) //最坏单链表的情况 最好平衡树的情况O(n * log n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/278655172/)

Solution-2：PrefixSum + Cache

1.遍历顺序依旧是dfs。

2.采用prefixSum的方法，避免进行过多的重复计算。
遍历到某个节点时，将从root到该节点这条路径的和累加，缓存起来，在后续过程，可以利用这些有用信息。

3.具体过程如下：
a.访问到每一个节点的时候，cur += node->val；
b.判断(cur - sum)有没有在字典中出现，有的话，答案累加其出现的频率（次数）
【原理1】：当遍历到当前节点node时，假设cur为x，也就是sum(root, node) = x。若字典中存在(x - sum)，说明在某个节点node1，sum(root, node1) = (x - sum)。
那么也就可以推断出来：sum(node1->next, node) = sum。也就是答案要求解的路径。
【原理2】：字典中存在的记录，一定是从root到node路径上所有节点的前缀和。也就是说保证了某个节点到当前节点node的【可通性】。因此，我们在遍历完每个节点后，要将其在字典中的“影响”消除。
c.将当前cur缓存到字典中，其出现频率加一。
至此，当前节点node“自身”考虑完成。

4.接下来递归考虑node的左儿子和右儿子。
当他们都考虑完成的时候，节点node才真正考虑完成。此时将cur在字典中出现的频率减一。
同时返回给父亲节点（回溯）

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/278714068/)
