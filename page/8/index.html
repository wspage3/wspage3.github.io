【DP专项】377.Combination Sum IV

一、题意

题目链接: 377.Combination Sum IV

1.Description

【输入】
给定一个长度为n的整数数组。数组元素都是正数。
给定一个正整数target。
【输出】
求出有多少种不同的组合方式，使得组合的sum等于target。
每个数字可以使用任意次。

2.Example

nums = [1, 2, 3]
target = 4

The possible combination ways are:
(1, 1, 1, 1)
(1, 1, 2)
(1, 2, 1)
(1, 3)
(2, 1, 1)
(2, 2)
(3, 1)

Note that different sequences are counted as different combinations.
Therefore the output is 7.

3.Corner Case

  1. 数组元素保证正数
  2. target保证正数
  3. 当n为0时，不能使得sum等于正数的target，所以答案为0

4.Follow Up

  1. 如果数组元素中有负数呢？

二、题解

Solution-1：Dynamic Programming(Iterative)

  1. 思考：
    Q: What is important so far???
    A: The sum

  2. 状态定义：dp[i]：代表有多少种不同的方式组成一个序列，使得序列和为i。

  3. 转移方程：dp[i] += dp[i - x]。其中(i - x)必须大于等于0；x是nums中出现的所有元素。
    解释：要想使得sum为i，那么一定是由(i - x)转移过来。
    举个例子：nums = [1, 2, 3]，target = 20。
    当计算到dp[10]的时候：10 = 9 + 1 = 8 + 2 = 7 + 3。
    所以：dp[10] = dp[9] + dp[8] + dp[7]

  4. 边界条件：dp[0] = 1。

  5. 求解方向：
    【Backward DP】
    i从1开始，到target结束。
    对于每个i，遍历nums所有的元素。如果(i - x)大于等于0，则dp[i]累加dp[i - x]。

【Forward DP】
已知dp[0] = 1。
i从0开始，到target - 1结束。因为最多只可能target - 1转移到target。
对于每个i，遍历nums所有的元素。如果(i + x)小于等于target，则dp[i + x]累加dp[i]。

  1. 空间优化：本题目无法进行空间降维。因为计算最后一项dp[target]的时候，可能使用到很多项之前的dp[i]。

  2. 注意：当用例为：[3,33,333]，10000时，可以看出dp[10000]其实为0，因为不管这么组合，都不可能使得sum为10000。
    此时在从1计算到10000的时候，存在某些i，使得dp[i]非常大，使用long long也不足以表示其方案数。
    此时，可以换成无符号的int来存放值。这样，当overflow的时候，unsigned int会自动处理溢出。而不会报Runtime Error。
    For example, unsigned int x = 4294967295 is ok but after do x += 1 operation, x will be 0 without any runtime error.

  3. Code(https://leetcode.com/submissions/detail/290501613/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)

  4. Code(https://leetcode.com/submissions/detail/290506960/)
    时间复杂度：O(n * target)
    空间复杂度：O(target)
