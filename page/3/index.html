【DP专项】279.Perfect Squares

发表于 2020-01-07

一、题意

题目链接: 279.Perfect Squares

1.Description

【输入】
给定一个正整数n；
【输出】
求出至少需要多少个”完全平方数”，使得它们的和等于n；
完全平方数：1, 4, 9, 16,…

2.Example

Input: n = 12
Output: 3
Explanation: 12 = 4 + 4 + 4.

Input: n = 13
Output: 2
Explanation: 13 = 4 + 9.

3.Corner Case

  1. 可以重复使用同一个数；
  2. n保证大于0；

二、题解

思路

  1. 回忆【322.Coin Change】，从数组中选出一些元素，使得其和为target，求最少使用几个元素。【可选可不选】【可选多次】【求最小个数】

  2. 本题几乎完全一样，数组为：[1, 4, 9, 16…]。

Solution-1：Dynamic Programming

  1. Q: What is important so far??? A: Cur value; Potential pre value; Pre cnt;

  2. 状态定义：dp[i]代表：最少需要多少个完全平方数，使其sum为i。

  3. 边界条件：dp[1] = 1。dp[0] = 0。（当i本身就是完全平方数的时候，此时dp[i] = 1 + dp[0] = 1）

  4. 转移方程：dp[i] = min(1 + dp[i - j * j])。其中1 <= j * j <= i。

  5. 计算顺序：
    【Backward-DP】
    当计算dp[i]的时候，需要使用到dp[i - j * j]的值。而(i - j * j)一定不超过i，所以从前往后计算；

  6. 空间优化：无法由一维线性降至常数；

  7. Code(https://leetcode.com/submissions/detail/261940601/)
    时间复杂度：O(n * sqrt(n))
    空间复杂度：O(n)

Solution-2：Math

  1. 四平方和定理(Lagrange’s four-square theorem)说明每个正整数均可表示为4个整数的平方和。
    所以本题目的答案就在[1,2,3,4]范围内。

  2. 如果n本身就是完全平方数，则答案为1：由它自身一个元素组成；

  3. The result is 4 if and only if n can be written in the form of 4^k(8m + 7).
    Please refer to Legendre’s three-square theorem.

  4. 考虑答案为2的情况：即由两个完全平方数组成。

  5. 其余情况，答案只剩3了。

  6. Code(https://leetcode.com/submissions/detail/264023785/)
    时间复杂度：O(sqrt(n))
    空间复杂度：O(1)

【DP专项】674.Longest Continuous Increasing Subsequence

发表于 2020-01-06

一、题意

题目链接: 674.Longest Continuous Increasing Subsequence

1.Description

【输入】
给定一个长度为n的整数数组nums；数组无序；
【输出】
求出最长递增子数组的长度。子数组要求连续。

2.Example

Input: [1,3,5,4,7]
Output: 3
Explanation: The longest continuous increasing subsequence is [1,3,5], its length is 3.
Even though [1,3,5,7] is also an increasing subsequence, it’s not a continuous one where 5 and 7 are separated by 4.

Input: [2,2,2,2,2]
Output: 1
Explanation: The longest continuous increasing subsequence is [2], its length is 1.

3.Corner Case

  1. 若n为0/1，直接返回0/1即可；
  2. 要求严格递增；

二、题解

思路

  1. 回忆【300.Longest Increasing Subsequence】，我们要求的是【最长递增子序列】的长度。
    现在要求的是【最长递增子数组】的长度。

  2. 前面可知：【子数组】（非空）的个数为：n * (n + 1) / 2；复杂度：O(n * n)

Solution-1：Brute Force

  1. 考虑以每个元素为开头的最长子数组。i：[0, n - 2]。
    对于一个固定的i，考虑其后面最长能接多少个元素。j：[i + 1, n - 1]。
    不管update最大值，最后返回即可。

  2. Code(https://leetcode.com/submissions/detail/291730890/)
    时间复杂度：O(n * n)
    空间复杂度：O(1)

Solution-2：Dynamic Programming

  1. Q: What is important so far??? A: Cur num; Pre num; Pre length;

  2. 状态定义：dp[i]表示：以nums[i]结尾的最大递增子数组的长度。

  3. 边界条件：dp[i] = 1;

  4. 转移方程：dp[i] = nums[i] > nums[i - 1] ? 1 + dp[i - 1] : 1;

  5. 计算顺序：
    【Backward-DP】
    i从1开始，到(n - 1)结束；

  6. 空间优化：从一维线性降至常数复杂度。

  7. Code(https://leetcode.com/submissions/detail/291732566/)
    时间复杂度：O(n)
    空间复杂度：O(n)

  8. Code(https://leetcode.com/submissions/detail/291732750/)
    时间复杂度：O(n)
    空间复杂度：O(1)

【DP专项】673.Number of Longest Increasing Subsequence

发表于 2020-01-06

一、题意

题目链接: 673.Number of Longest Increasing Subsequence

1.Description

【输入】
给定一个长度为n的整数数组nums；数组无序；
【输出】
求出LIS(最长递增子序列)的个数。序列不同于子数组，不要求连续。

2.Example

Input: [1,3,5,4,7]
Output: 2
Explanation: The two longest increasing subsequence are [1, 3, 4, 7] and [1, 3, 5, 7].

Input: [2,2,2,2,2]
Output: 5
Explanation: The length of longest continuous increasing subsequence is 1, and there are 5 subsequences’ length is 1, so output 5.

3.Corner Case

  1. 若n为0/1，直接返回0/1即可；
  2. 要求严格递增；

二、题解

思路

  1. 回忆【300.Longest Increasing Subsequence】，我们要求的答案是LIS的长度。
    现在要求的是LIS的方案数。

Solution-1：Dynamic Programming

  1. Q: What is important so far??? A: Cur num; Pre num; Pre length; Cnt of Pre length;

  2. 状态定义：
    dp[i]表示：以nums[i]结尾的递增子序列的最大长度。
    cnt[i]表示：以nums[i]结尾的最大递增子序列（长度为res[i]），对应的方案数。

  3. 边界条件：
    dp[i]初始化为1；
    cnt[i]初始化为1；

  4. 转移方程：
    dp[i] = max(dp[i], dp[j] + 1)。其中：0 <= j <= i - 1；且nums[i] > nums[j]；

  5. 计算顺序：
    【Backward-DP】
    i从1开始，到(n - 1)结束；
    对于一个确定的nums[i]，j在[0, i - 1]上遍历，找出所有nums[j] < nums[i]的情况：
    若dp[j] >= dp[i]：此时，可以将nums[i]接在nums[j]后，构成长度为(dp[j] + 1)的LIS。此时cnt[i] = cnt[j]。
    若dp[j] == dp[i] - 1：此时，虽然LIS的长度不会变大，但方案数增加了。cnt[i] += cnt[j]。
    若dp[j] < dp[i] - 1：此时，LIS长度不会变大，不考虑这种情况。cnt[i]保持不变。

  6. 空间优化：无法由一维线性降至常数：计算nums[i]的时候，需要考虑前面所有的元素。

  7. Code(https://leetcode.com/submissions/detail/291708328/)
    时间复杂度：O(n ^ 2)
    空间复杂度：O(n)

【DP专项】091.Decode Ways

发表于 2020-01-04 更新于 2020-01-06

一、题意

题目链接: 091.Decode Ways

1.Description

【输入】
给定一个非空的字符串。字符串所有字符都是数字，即’0’~’9’；
【输出】
求出该字符串有多少种解码方式。编码规则如下：
‘A’ -> 1
‘B’ -> 2
…
‘Z’ -> 26

2.Example

Input: “226”
Output: 3
Explanation: It could be decoded as “BZ” (2 26), “VF” (22 6), or “BBF” (2 2 6).

3.Corner Case

  1. 字符串非空，即n > 0；
  2. 若加密的串不合法的时候，无法找到一种方法解密，答案为0；

二、题解

Solution-1：Dynamic Programming

  1. Q: What is important so far? A: Cur character; Pre character; How many ways before?;

  2. 状态定义：dp[i]代表[0, i - 1]范围内，一共有多少种解码方式？答案即为dp[n]；

  3. 边界条件：
    a. 若字符串以’0’为开头；或者字符串中间出现了类似’xxx30xxx’、’xxx60xxx’这样的情况，则直接返回0；
    b. dp[1]：代表遍历到s[0]后，可能的组合方式。
    c. dp[0]：为方便后续计算，初始化为1。

  4. 转移方程：
    a. 依次考虑s[1]s[n - 1]。将相应的结果存放在dp[2]dp[n]；即：i:[1, n-1]，j:[2, n]
    b. 若s[j]为’0’：同时s[j - 1]是’1’或者’2’，则此时j只能和j-1组合：dp[i] = dp[i - 2]；
    c. 若s[j - 1]是’1’，s[j]非’0’：
    或s[j - 1]是’2’，s[j]在[1, 6]范围内，则此时j-1和j既能单独，也可以结合：
    dp[i] = dp[i - 2]；
    d. 其它情况的话，j-1和j只能单独：dp[i] = dp[i - 1]；

  5. 计算顺序：
    【Backward】
    从字符串的第2个字符开始（index:1），计算到字符串的最后一个字符结束（index:n - 1）。

  6. 空间优化：一维线性空间压缩为常数空间。

  7. Code(https://leetcode.com/submissions/detail/291568499/)
    时间复杂度：O(n)
    空间复杂度：O(n)

  8. Code(https://leetcode.com/submissions/detail/272543841/)
    时间复杂度：O(n)
    空间复杂度：O(1)

【DP专项】338.Counting Bits

发表于 2020-01-03

一、题意

题目链接: 338.Counting Bits

1.Description

【输入】
给定一个非负整数n。
【输出】
求出范围[0, n]内，所有元素的二进制形式中1的个数。以数组的形式返回（数组长度为：n + 1）。

2.Example

Input: 5
0: 0000
1: 0001
2: 0010
3: 0011
4: 0100
5: 0101
Output: [0,1,1,2,1,2]

3.Corner Case

  1. 当n为0时，返回长度为1的数组。[0]
  2. 要求空间复杂度O(n)
  3. 要求时间复杂度O(n)，而非O(n * l)，其中l是整数的比特数。4字节时，l = 32。
  4. 要求不适用内置函数，如：__builtin_popcount(n)

二、题解

Solution-1：Brute Force

  1. 依次遍历[0, n]，单独求出每个数字的二进制表示方式中1的个数。

  2. 对于任一数字，让其不断除以2，计算最低位1的个数。

  3. Code(https://leetcode.com/submissions/detail/282685606/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

  4. Code(https://leetcode.com/submissions/detail/282686107/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

  5. Code(https://leetcode.com/submissions/detail/282686317/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

Solution-2：Dynamic Programming

  1. 考虑1100000000001这个数字，其十进制为：6145。
    按照Solution-1，我们对这个数字每次除以2（右移一位），需要处理13次。
    而其中有10个0，3个1。有没有办法跳过这些0呢？

  2. temp不再右移一位。而是：temp = temp & (temp - 1);
    继续上面的例子：
    1100000000001：计数一次
    1100000000000：计数一次
    1000000000000：计数一次
    0000000000000：结束
    可以看到，我们只处理了3次，也就是说通过这样的方法，对于任意数字x。可以将时间复杂度降低为O(k)，k是其中1的个数。

  3. Code(https://leetcode.com/submissions/detail/282687122/)
    时间复杂度：O(n * k) //k是数字的二进制表示形式下，其中1的个数。
    空间复杂度：O(n)

Solution-3：Dynamic Programming

  1. 根据Solution-2的启示：我们可以发现：res[i] = res[i & (i - 1)] + 1;

  2. Code(https://leetcode.com/submissions/detail/282688195/)
    时间复杂度：O(n)
    空间复杂度：O(n)
