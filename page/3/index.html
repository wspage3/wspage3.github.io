20.Valid Parentheses

一、题意

题目链接: 20.Valid Parentheses

1.Description

给定一个长度为n的字符串s；
判断该字符串是否一个合法的括号表达式；
括号有三种：[] {} ()
open的bracket最终必须有一个同类型close的bracket与之匹配；
且上述匹配的顺序必须是正确的。

2.Example

Input: “()[]{}”
Output: true

Input: “([)]”
Output: false

3.Corner Case

n: 当n为0的时候，也认为是合法的；

二、题解

Solution-1：Stack

1.若n是奇数，那么一定非法。
若n是0，则直接return true。

2.考虑n为大于0的偶数：
模拟括号匹配的过程：i在范围[0, n - 1]内遍历：
先将s[0]压入栈；
考察s[1]，若s[1]和s[0]是同类型的，且一个s[1]是close，s[0]是open的，那么说明这两个是一对。
则此时将s[0]从栈中抛出，i跳到位置2，考虑第二个元素。
若s[1]的出现无法和s[0]匹配掉，则将s[1]在压入栈，考虑下一个元素。
……
直到所有的n个元素都遍历过了，此时检查栈，如果栈是空的，说明所有的括号都可以匹配。
如果此时栈非空，则说明存在元素无法匹配。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/285447709/)

Solution-2：Stack

1.另一种方法：
从[0, n - 1]考虑每一个s[i]
若s[i]是open的bracket，则将其“另一半”压入栈。
否则的话，判断此时栈是否为空，是的话，直接return false。
栈不空的话，判断当前元素是否和栈顶元素相同，不同的话，也直接return false。
相同的话，将栈顶元素抛出，转移到考虑s[i + 1]

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/285450396/)

22.Generate Parentheses

一、题意

题目链接: 22.Generate Parentheses

1.Description

给定一个整数n，代表有n对括号。这是一对：( )
求出这n对括号所有可能的组合方式，要求组合后是一个有效的括号表达式。
即：匹配的一对括号，左边是(，右边是)

2.Example

n = 3
[
“((()))”,
“(()())”,
“(())()”,
“()(())”,
“()()()”
]

3.Corner Case

n: 当n为0的时候，直接返回空；

二、题解

Solution-1：Backtracking(Recursive)

1.给定的n，代表有n个open bracket，n个close bracket。

2.对于任意时刻，我们仅可选一个bracket，要么是open的，要么是close的。
只要leftCnt大于0，就可以选择open的。
而选择close的，需要rightCnt大于leftCnt，以保证结果先左后右的合法性。

3.当leftCnt, rightCnt的值都为0时，此时构造出来的string s就是一个合法的答案。将其添加到答案中。
并且返回。（回溯）

4.递归树的某一个子节点计算完成后返回了，其父节点进行考虑，若当前还有其他的儿子，则递归调用其他的儿子。
对于一个节点来说，其左儿子可以认为是“选择open bracket”，其右儿子可以认为是“选择close bracket”

时间复杂度：O() //to do
空间复杂度：O()

17.Letter Combinations of a Phone Number

一、题意

题目链接: 17.Letter Combinations of a Phone Number

1.Description

给定一个长度为n的字符串。
该字符串的每一个元素都是：[2, 9]的整数。
对应的，手机键盘上的九键输入法。如2可以代表：[a, b, c]。
求出给定的数字序列，所有可能的字符串。

2.Example

Input: “23”
Output: [“ad”, “ae”, “af”, “bd”, “be”, “bf”, “cd”, “ce”, “cf”].

3.Corner Case

n: 当n为0的时候，直接返回空；
尽管example中输出的是字典序，但本题目不要求严格的字典序输出。

二、题解

Solution-1：Brute Force

1.对于长度为n的序列，依次考虑每一个数字。

2.对于[2, 3]来说：
先考虑2，由于2可以代表[a, b, c]，则将[a, b, c]放入ans中。
再考虑3，由于3可以代表[d, e, f]，则对于ans中的所有元素，分别将d/e/f拼接在其后面。
即：[ad, bd, cd, ae, be, ce, af, bf, cf]

3.所以说，首先外层循环需要考虑n次。
对于每个n来说，需要考虑其所有代表的字母。记x次。则x要么是3，要么是4。
对于每个代表的字母，需要考虑当前ans中的个数：然后将自己加入到这些已有的ans后面。
由于这个过程需要x次，所以可以用一个temp数组，都存起来。当遍历完x次，将ans更新为temp即可。

4.对于将temp赋值给ans的操作，可以：
ans = temp;//vector支持直接赋值过去。
参考：https://zh.cppreference.com/w/cpp/container/vector/operator%3D
ans.swap(temp)//O(1)的复杂度，更快。因为swap只交换指针，而不替换数据。
参考：http://www.cplusplus.com/reference/vector/vector/swap/
https://www.zhihu.com/question/310462377

时间复杂度：O(3 ^ n)
空间复杂度：O(1)

Code(https://leetcode.com/submissions/detail/285392928/)

48.Rotate Image

一、题意

题目链接: 48.Rotate Image

1.Description

给定一个n阶矩阵。
将这个矩阵顺时针旋转90°。
不得申请额外空间。即：in-place就地操作。

2.Example

Given input matrix =
[
[1,2,3],
[4,5,6],
[7,8,9]
],
rotate the input matrix in-place such that it becomes:
[
[7,4,1],
[8,5,2],
[9,6,3]
]

3.Corner Case

n: 当n为0的时候，直接返回false；

二、题解

Solution-1：Layer By Layer，One By One

1.n阶矩阵，需要考虑n / 2层，即层数：[0, n / 2 - 1]
2.对于每一层，j：[i, n - 2 - i]

时间复杂度：O(n ^ 2)
空间复杂度：O(1)

reverse后：

[7,8,9],

153.Find Minimum in Rotated Sorted Array

一、题意

题目链接: 153.Find Minimum in Rotated Sorted Array

1.Description

给定一个长度为n的整数数组nums；
该数组原本是升序的，结果在某个支点处旋转了一次，如：
i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]
假设数组中不存在相同的数字。
找出这个数组中最小的值。

2.Example

Input: [3,4,5,1,2]
Output: 1

3.Corner Case

n: 当n为0的时候，直接返回false；

二、题解

Solution-1：Brute Force

1.常规思路：从前到后遍历一遍nums，找到最小值返回。

时间复杂度：O(n)
空间复杂度：O(1)

Solution-2：Binary Search

1.观察：[0,1,2,4,5,6]
经过rotate后，可能存在以下几种不同情况：
[1,2,4,5,6, 0]
[2,4,5,6, 0,1]
[4,5,6, 0,1,2]
[5,6, 0,1,2,4]
[6, 0,1,2,4,5]

2.考虑寻找[left, right]中的最小值。
若：nums[left] < nums[right]，则说明[left, right]范围内完全升序。nums[left]就是答案。
否则：
mid = left + (right - left) / 2;
若nums[mid] > nums[right]，则说明，这个区间中间某个值比nums[right]还要大，说明mid此时在左侧。
而最小值一定在右侧[mid + 1, right]。
若nums[mid] < nums[right]，则说明，[mid, right]这个范围是有序的。要寻找的最小值，不可能在mid右侧，一定在包含其的左侧，即：[left, mid]。
若nums[mid] == nums[right]，不可能的……

3.用二分的方法，每次把结果可能出现的区间缩小一半。
最终当区间长度为2的时候，left = x, right = x + 1, mid = x
判断nums[mid]和nums[right]的大小关系，也就是判断左边元素和右边元素的大小。
若nums[mid]大于nums[right]，即答案一定在[mid + 1, right]中，也就是右边元素本身。
若nums[mid]小于nums[right]，即答案一定在[left, mid]中，也就是左边元素本身。
循环结束后，left一定等于right。此时返回left的值即可。

时间复杂度：O(log n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/285160090/)

33. Search in Rotated Sorted Array

一、题意

题目链接: 33. Search in Rotated Sorted Array

1.Description

给定一个长度为n的整数数组nums；
该数组原本是升序的，结果在某个支点处旋转了一次，如：
i.e., [0,1,2,4,5,6,7] might become [4,5,6,7,0,1,2]
给定一个整数target；
判断nums中是否包含该target，如果是，返回其index；否则，返回-1。
假设数组中所有数字都不相同。
要求时间复杂度为：O(log n)

2.Example

Input: nums = [4,5,6,7,0,1,2], target = 0
Output: 4

3.Corner Case

n: 当n为0的时候，直接返回-1；

二、题解

Solution-1：Brute Force

1.常规思路：从前到后遍历一遍nums，如果找到target则return。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/284976005/)

Solution-2：Binary Search

1.题目要求time complexity为O(log n)，则第一时间想到了Binary Search

2.nums由于旋转的原因，虽说整体无序。但有两部分都是升序排列的。

3.例如原有序数组为：[2,3,5,7,8]，经过rotate后所有可能为：
[3,5,7,8,2]
[5,7,8,2,3]
[7,8,2,3,5]
[8,2,3,5,7]
可能结果有(n - 1)种。
仔细观察可以发现，不管rotate成哪一种，都可以找出左边、右边两部分，它们各自有序。
而且：【右边子数组的所有元素一定比左边子数组的第一个元素小；】

4.举个栗子：
对于[3,5,7,8,2]来说，left置为0，right置为4
第一次：mid = 2, nums[mid] = 7,
此时若target在范围[3, INT_MAX]内，即target不可能在另外一侧，因为另外一侧所有的元素都比nums[0]要小。
直接进行二分搜索即可，用来比较的值就是nums[mid]。
此时若target不在范围[3, INT_MAX]内，说明：target不可能在左侧。需要将其在右侧搜索。即left往右移动。

5.判断target和nums[mid]一个简单明了的方法就是：看他们俩是否都同时比nums[0]大或者比nums[0]小。
如果都比nums[0]大的话，说明这两个数都在左侧。
如果都比nums[0]小的话，说明这两个数都在右侧。
如果一个在左侧，一个在右侧的话，说明当前的nums[mid]对于在另外一侧寻找到target没有参考意义。
此时判断target和nums[0]的关系：
若target大于nums[0]，说明target此时在左侧，想办法将right指针左移。就是: num = INT_MAX;相当于认为右侧所有数都是INT_MAX
若target小于nums[0]，说明target此时在右侧，想办法将left指针右移。就是: num = INT_MIN;相当于认为左侧所有数都是INT_MIN

时间复杂度：O(log n)
空间复杂度：O(1)

一、题意

题目链接: Leetcode309-Best Time to Buy and Sell Stock with Cooldown

1.Description

给定一个长度为n的整数数组prices；prices[i]代表第i天股票的价格。
最多交易次数不限，求出你最多能赚多少钱。
前提是：买入下一股的时候，必须将上一股卖掉。
而且建议冷却期为1天，即：今天卖掉，后天才能买入。

2.Example

Input: [1,2,3,0,2]
Output: 3
Explanation: transactions = [buy, sell, cooldown, buy, sell]

3.Corner Case

n: 当n为0/1的时候，直接返回0；
赚不到钱则返回0；

二、题解

Solution-1：Dynamic Programming

buy[i] = max(buy[i - 1], sell[i - 2] - prices[i]);//左边代表：今天休息；右边代表：今天用前天的利润来进一批货。
sell[i] = max(sell[i - 1], buy[i - 1] + prices[i]);//左边代表：今天休息；右边代表：今天在昨天进货的基础上，卖掉货。
buy[i]:时间范围[0, i]内，最后一次操作是buy或者rest，所能获得的最高利润
sell[i]:时间范围[0, i]内，最后一次操作是sell或者rest，所能获得的最高利润

Leetcode714-Best Time to Buy and Sell Stock with Transaction Fee

一、题意

题目链接: Leetcode714-Best Time to Buy and Sell Stock with Transaction Fee

1.Description

给定一个长度为n的整数数组prices；prices[i]代表第i天股票的价格。
非负数fee代表每次交易需要的费用。
最多交易次数不限，求出你最多能赚多少钱。
前提是：买入下一股的时候，必须将上一股卖掉。

2.Example

Input: prices = [1, 3, 2, 8, 4, 9], fee = 2
Output: 8
Explanation: The maximum profit can be achieved by:
Buying at prices[0] = 1
Selling at prices[3] = 8
Buying at prices[4] = 4
Selling at prices[5] = 9
The total profit is ((8 - 1) - 2) + ((9 - 4) - 2) = 8.

3.Corner Case

0 < prices.length <= 50000.
0 < prices[i] < 50000.
0 <= fee < 50000.

n: 当n为0/1的时候，直接返回0；
赚不到钱则返回0；

二、题解

Solution-1：Dynamic Programming

1.根据leetcode123、188，求出状态转移方程。
由于不考虑交易次数的限制，而加上了考虑fee，所以可以得出以下方程：
dp[i] = max(dp[i - 1], dp[j] + prices[i] - prices[j] - fee);
= max(dp[i - 1], prices[i] - fee - (prices[j] - dp[j]));
dp[i]含义为：范围[0, i]内，所能获取的最高利润
i:[1, n - 1] dp[0] = 0
j:[0, i - 1] 当j为i的时候，相当于今天买了，今天卖，肯定亏一次手续费。

2.所以i在[1, n - 1]内循环，对于每个固定的i，在[0, i - 1]中找出一个j，使得表达式”(prices[j] - dp[j])”的值最小。
找出来j的位置后，dp[i]就可以求出；结果返回dp[n - 1]即可。

时间复杂度：O(n ^ 2)
空间复杂度：O(n)

结果和预期一样，TLE

3.Code(https://leetcode.com/submissions/detail/284928102/)

Solution-2：Dynamic Programming + Optimization

1.首先，由于i每次递增1：
对于当前的i来说，首要任务是在[0, i - 1]找出目标表达式”(prices[j] - dp[j])”的最小值。
而前一轮的i - 1，已经计算出了[0, i - 2]内目标表达式的最小值。
所以对于当前i来说，直接复用该结果，可以使得O(n)的time complexity降低到O(1)。

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/284929795/)

3.由于dp[i]只需要知道dp[i - 1]的值就可以进行计算了。所以可以使得O(n)的space complexity降低到O(1)。

时间复杂度：O(n)
空间复杂度：O(1)

Leetcode188-Best Time to Buy and Sell Stock IV

一、题意

题目链接: Leetcode188-Best Time to Buy and Sell Stock IV

1.Description

给定一个长度为n的整数数组prices；prices[i]代表第i天股票的价格。
如果你最多能交易k次，即：买k次，卖k次。求出你最多能赚多少钱。
前提是：买入下一股的时候，必须将上一股卖掉。

2.Example

Input: [2,4,1], k = 2
Output: 2
Explanation: Buy on day 1 (price = 2) and sell on day 2 (price = 4), profit = 4-2 = 2.

3.Corner Case

n: 当n为0/1的时候，直接返回0；
k: 当k <= 0的时候，直接返回0；
当k >= n的时候，将k赋值为n - 1；
赚不到钱则返回0；

二、题解

Solution-1：Dynamic Programming

1.根据leetcode123，可以得到相同的状态转移方程：
dp[j][i] = max(dp[j][i - 1], dp[j - 1][m] + prices[i] - prices[m]);
其中：j:[1, k] i:[0, n - 1] m:[0, i]

2.申请(k + 1) * n的dp数组空间，答案就是dp[k][n - 1];
j从[1, k]遍历
i从[1, n - 1]遍历
m从[0, i]遍历
对于每次固定好的j和i，找出一个位置m，使得prices[m] - dp[j - 1][m]的值最小。

时间复杂度：O(k * n ^ 2)
空间复杂度：O(k * n)

3.Code(https://leetcode.com/submissions/detail/284743219/)

Solution-2：Dynamic Programming + Optimization

1.从Solution-1的解法来看，对于当前(k, i)序列，假设通过O(n)的循环，找到了一个位置m1，使得表达式的值最小了，从而算出了当前(k, i)的答案。
接着迭代到当前(k, i + 1)序列，此时还需要O(n)的循环：找到当前序列一个位置m2，使得表达式的值最小。
由于我们在上一次循环中已经计算出了[0, i]范围内使得表达式最小的一个位置。当前(k, i + 1)序列，直接通过O(1)的时间就可以算出来，而不必继续遍历[0, i + 1]

2.所以，对于遍历当前i位置来说，可以直接访问上一次i - 1的计算结果。从而把整体的时间复杂度优化为O(k * n)

时间复杂度：O(k * n)
空间复杂度：O(k * n)

1.尝试调换两次循环的次序
2.进一步优化空间，将二维降成一维
