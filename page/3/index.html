【DP】【Two Pointers】392.Is Subsequence

发表于 2020-01-17

一、题意

题目链接: 392.Is Subsequence

1.Description

【输入】
给定两个字符串s、t；长度分别为n1、n2；字符串中字符均为小写字母；
【输出】
判断s是否为t的某个子序列(subsequence)；

2.Example

s = “abc”, t = “ahbgdc”
Return true.

s = “axc”, t = “ahbgdc”
Return false.

3.Corner Case

  1. t长度很大，n2 <= 500000
  2. s长度较小，n1 <= 100
  3. 若n1 == 0，不管n2是否为0，答案为true；
  4. 若n1 > n2，则答案为false；

4.Follow Up

假设有n个s，需要判断一个一个判断其是否为t的子串，此时，代码需要怎样修改？

二、题解

Solution-1：Straightforward Solution(Two Pointers)

  1. 初始化：i指向s[0]，j指向t[0]。
    过程：当t还没遍历完的时候：
    a. 当s[i] == t[j]的时候，i后移一位。同时判断s是否已经遍历完了，如果遍历完了，则return true；
    b. j往后移动一位；
    结束：当t已经遍历完了，如果此时s还没遍历完，则return false；

  2. Code(https://leetcode.com/submissions/detail/294917783/)
    时间复杂度：O(len(t))
    空间复杂度：O(1)

  3. Code(https://leetcode.com/submissions/detail/294771053/)
    时间复杂度：O(len(t))
    空间复杂度：O(1)

Solution-2：Dynamic Programming

  1. 回忆【1143.Longest Common Subsequence】，我们用DP的方法找出两个字符串s1，s2的LCS。
    如果s1是s2的subsequence，那么二者的LCS就是s1；

  2. Code(https://leetcode.com/submissions/detail/294934725/)
    时间复杂度：O(n1 * n2)
    空间复杂度：O(n1 * n2)

【DP专项】1143.Longest Common Subsequence

发表于 2020-01-16

一、题意

题目链接: 1143.Longest Common Subsequence

1.Description

【输入】
给定两个字符串text1，text2，长度分别为n1，n2；
【输出】
找出text1和text2的LCS(Longest Common Subsequence)的长度；
LCS：最长公共子序列
如果没有LCS，则返回0；

2.Example

Input: text1 = “abcde”, text2 = “ace”
Output: 3
Explanation: The longest common subsequence is “ace” and its length is 3.

Input: text1 = “abc”, text2 = “def”
Output: 0
Explanation: There is no such common subsequence, so the result is 0.

3.Corner Case

  1. 若n1 == 0 || n2 == 0，则答案为0；

二、题解

思路

  1. 若采用暴力法，则时间复杂度为指数形式的。

Solution-1：Dynamic Programming

  1. 状态定义：dp[i][j]代表：text1的子串[0, i - 1]，和text2的子串[0, j - 1]，它们两者的LCS的长度。

  2. 边界条件：为方便转移方程的统一性：即当i或者j为0的时候，需要考虑i - 1以及j - 1为负数的情况。
    将dp数组开到：(n1 + 1) * (n2 + 1)大小。当i为0，或者j为0时，元素初始化为0。

  3. 转移方程：
    a. 若text1[i - 1] == text2[j - 1]，则dp[i][j] = 1 + dp[i - 1][j - 1];
    b. 若text1[i - 1] != text2[j - 1]，则dp[i][j] = max(dp[i][j - 1], dp[i - 1][j]);

  4. 计算顺序：
    【Backward-DP】
    i从1开始，到n1结束；因为计算第i行的时候，需要使用到第i - 1行的元素；
    j从1开始，到n2结束；因为计算第i行，第j列的时候，需要使用到第i行，第j - 1列的元素；

  5. Code(https://leetcode.com/submissions/detail/294723307/)
    时间复杂度：O(n1 * n2)
    空间复杂度：O(n1 * n2)

  6. 空间优化：pre变量保存左上角的元素。

  7. Code(https://leetcode.com/submissions/detail/294725696/)
    时间复杂度：O(n1 * n2)
    空间复杂度：O(min(n1, n2))

【DP专项】516.Longest Palindromic Subsequence

发表于 2020-01-16

一、题意

题目链接: 516.Longest Palindromic Subsequence

1.Description

【输入】
给定一个长度为n的字符串s。
【输出】
找出一个s的一个子序列，该子序列构成回文串，且长度最大。

2.Example

Input: “bbbab”
Output: 4
One possible longest palindromic subsequence is “bbbb”.

Input: “cbbd”
Output: 2
One possible longest palindromic subsequence is “bb”.

3.Corner Case

  1. 当n <= 1时，答案为n；
  2. n <= 1000；

二、题解

思路

  1. 回忆【005.Longest Palindromic Substring】，我们找出任意一个s的最长回文子数组。
    而本题要求的是s的最长回文子序列的长度。

  2. 回忆【300.Longest Increasing Subsequence】中，有如下分析：
    【子序列】不要求连续，所以相较于【子数组】来说，数目更多。
    对于一个长度为n的数组来说：
    【子数组】（非空）的个数为：n * (n + 1) / 2；复杂度：O(n * n)
    【子序列】（非空）的个数为：(2 ^ n) - 1；复杂度：O(2 ^ n)

  3. 如果采用暴力方法的话，时间复杂度为O(n * (2 ^ n))；

Solution-1：Dynamic Programming

  1. Q: What is important so far??? A: Cur char(end); Start char(start); Pre length;

  2. 状态定义：dp[i][j]代表：在范围[i, j]内，最长的回文子序列的长度。答案即：dp[0][n - 1]；

  3. 边界条件：
    a. i == j时，dp[i][j] = 1；
    b. i > j时，dp[i][j]无意义；

  4. 转移方程：
    a. 若s[i] == s[j]，则dp[i][j] = 2 + dp[i + 1][j - 1]；
    考虑dp[i + 1][j - 1]的合法性：(i + 1 > j - 1时，dp[i + 1][j - 1]为0)
    b. 若s[i] != s[j]，则dp[i][j] = max(dp[i][j - 1], dp[i + 1][j])；
    不需要考虑dp[i][j - 1]和dp[i + 1][j]的合法性，画图：二维数组即可看出。

  5. 计算顺序：
    【Backward-DP】
    从上面的状态转移方程中可以看出：
    计算当前row的时候(i)，需要使用到下一row的内容(i + 1)。所以i从n - 1开始，到0结束；
    计算当前col的时候(j)，需要使用到前一col的内容(j - 1)。所以j从i开始，到n - 1结束；

  6. Code(https://leetcode.com/submissions/detail/294675718/)
    时间复杂度：O(n * n)
    空间复杂度：O(n * n)

  7. 空间优化：由于计算第(i)行的时候，只需要用到第(i + 1)行的数据，所以可以优化为O(n)的空间复杂度。
    具体过程：
    a. 在计算dp[i][j]的时候，需要用到它左下角的元素、左边的元素、下边的元素。
    b. 考虑dp数组压缩为一维的情况：左边的元素(dp[j - 1])、下边的元素(dp[j])。
    c. 由于在计算出来dp[j - 1]的时候，其之前的值已被覆盖。而被覆盖的该值，对于dp[j]来说，就是其左下角的元素。
    d. 所以我们在计算dp[j]的时候，先将其已存的值保存到prev变量中。在计算dp[j + 1]的时候，直接使用prev变量即可，计算后再更新prev变量。

  8. Code(https://leetcode.com/submissions/detail/294680816/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)
