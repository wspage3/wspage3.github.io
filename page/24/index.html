【二叉树专项】105.Construct Binary Tree from Preorder and Inorder Traversal

发表于 2019-12-27

一、题意

题目链接: 105.Construct Binary Tree from Preorder and Inorder Traversal

1.Description

【输入】
给定两个二维数组，分别代表一棵二叉树的：
前序遍历序列
中序遍历序列
【输出】
求出这棵二叉树。

2.Example

Input:
preorder = [3,9,20,15,7]
inorder = [9,3,15,20,7]

Output: [3,9,20,NULL,NULL,15,7]

3.Corner Case

  1. 目标数组为空时，树为NULL。
  2. 目标数组长度为1时，树为单节点。

二、题解

Solution-1：Recursive Solution(Postorder)

1.根据以下特性：
preorder序列的第一个数字一定是：root的值
inorder序列：root的值左边都是root的左子树，右边都是root的右子树。

2.对于任意任意两个序列：build(preStart, preEnd, inStart, inEnd)
a.根节点root一定是preorder[preStart]
b.在inorder中找到root的位置，记为pos。则左子树的节点数目：pos - inStart
c.递归构建左子树：build(preStart + 1, preStart + (pos - inStart), inStart, pos - 1)
d.递归构建右子树：build(preStart + (pos - inStart) + 1, preEnd, pos + 1, inEnd)
e.将c.d两步的返回值赋值为root的左子树和右子树，return给上一层。

时间复杂度：O(n)
空间复杂度：O(n)

【二叉树专项】116.Populating Next Right Pointers in Each Node

发表于 2019-12-27

一、题意

题目链接: 116.Populating Next Right Pointers in Each Node

1.Description

【输入】
给定一棵完美二叉树(Perfect Binary Tree)。
完美二叉树：树中所有的叶子节点在同一层，每个非叶子节点都有两个儿子。
树的定义如下：

除了常规的：value，左儿子，右儿子外，还有一个成员是：next指针，指向其右侧的节点。
所有节点的next指针初始化为NULL。
【输出】
求出所有节点的next指针，并保存在当前的树中。
若某个节点右侧没有节点了，则值保持为NULL就行。

2.Example

Input: root = [1,2,3,4,5,6,7]
Output: [1,#,2,3,#,4,5,6,7,#]
Explanation: Given the above perfect binary tree (Figure A), your function should populate each next pointer to point to its next right node, just like in Figure B. The serialized output is in level order as connected by the next pointers, with ‘#’ signifying the end of each level.

3.Corner Case

  1. 树为NULL时，无需操作。
  2. 树为单节点时，无需操作。
  3. 要求O(1)的空间复杂度。允许使用递归，假定由递归产生的space开销不计入空间复杂度中。

二、题解

Solution-1：Iterative Solution

1.temp指针垂直方向迭代：
从root（第1层）出发，到倒数第二层为止。
代表每一层的最左边节点。

2.cur指针水平方向迭代：
对于每一层来说，cur从最左边开始，到最右边为止。
代表每一层的任意非空节点。

3.遍历到每个节点时，将其：
左儿子的next指针设置为：cur->left->next = cur->right;
右儿子的next指针设置为：cur->right->next = cur->next == NULL ? NULL : cur->next->left;

4.循环结束时，所有的节点的next指针都被设置过了。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/278169311/)

Solution-2：Recursive Solution(Preorder)

1.遍历顺序为preorder：先处理完自身，递归其左右儿子时，才能顺利往下进行。

时间复杂度：O(n)
空间复杂度：O(1)

2.Code(https://leetcode.com/submissions/detail/278170494/)

【BST专项】173.Binary Search Tree Iterator

发表于 2019-12-26

一、题意

题目链接: 173.Binary Search Tree Iterator

1.Description

【输入】
给定一棵BST。
【输出】
要求实现两个BST相关迭代器：
1.int next()：返回这棵BST中下一个最小的数字
2.bool hasNext()：判断这棵BST中是否有一个最小的数字

2.Example

Input: [7,3,15,NULL,NULL,9,20]

BSTIterator iterator = new BSTIterator(root);
iterator.next(); // 返回 3
iterator.next(); // 返回 7
iterator.hasNext(); // 返回 true
iterator.next(); // 返回 9
iterator.hasNext(); // 返回 true
iterator.next(); // 返回 15
iterator.hasNext(); // 返回 true
iterator.next(); // 返回 20
iterator.hasNext(); // 返回 false

3.Corner Case

  1. 当遍历到BST的最后一个节点时，hasNext()返回false。
  2. 题目保证next调用合法，所以不考虑该情况。

二、题解

要求

  1. next() 和 hasNext() 操作的时间复杂度是 O(1)，并使用 O(h) 内存，其中 h 是树的高度。
  2. 可以假设 next() 调用总是有效的，也就是说，当调用 next() 时，BST 中至少存在一个下一个最小的数。

Solution-1：

1.根据题意，next()和hasNext()要求o(1)的time和o(h)的space

2.第一想法是：把BST的节点都inorder读取出来，放在数组中，这样很好实现上述两个函数，但是o(n)的space不满足。

3.思路：要想依次获取BST中最小的节点，首先从root开始，一路往左下走，当某个节点N没有左儿子了，该节点就是最小的

4.现在要接着找第二小的节点：假设F是N的父亲，F一定大于N；
由于N没有左儿子，考虑N的右儿子，如果没有的话，则F就是第二小的节点；
如果有的话，假设为M，以M为根的所有节点一定都大于N，且小于F，则以M开始，继续一直往左下走。。。。。。
一直往左下走的动作，可以利用stack，当调用next()的时候，栈顶的就是最小的元素，把它pop出来，同时以其右儿子为root，继续往左下深入，再入栈

5.hasNext()，显然满足time和space复杂度要求；
next()，该函数最多被调用n次，n是BST的节点个数。pushAll(temp->right)同样最多调用n次，平均下来就是o(1)的时间复杂度了。
而任一时刻，stack的大小都不会超过log n，满足空间复杂度。

时间复杂度：O(1)
空间复杂度：O(h)

6.Code(https://leetcode.com/submissions/detail/278065392/)

2.现在我们要找出所有从root到leaf的路径，保存在一维字符串数组中。

时间复杂度：O(n)

【二叉树专项】236.Lowest Common Ancestor of a Binary Tree

发表于 2019-12-26

一、题意

题目链接: 236.Lowest Common Ancestor of a Binary Tree

1.Description

【输入】
给定一棵二叉树。
给定两个节点p、q。
【输出】
求出p、q的最近公共祖先LCA(Lowest Common Ancestor)。
LCA：对于节点p、q来说，若T都是它们的祖先，而且是距离p、q最近的一个。则T就是节点p、q的LCA。

2.Example

Input: root = [3,5,1,6,2,0,8,null,null,7,4], p = 5, q = 4
Output: 5
Explanation: The LCA of nodes 5 and 4 is 5, since a node can be a descendant of itself according to the LCA definition.

3.Corner Case

  1. 所有节点的value不重复。
  2. p和q是两个不同的node，且都是存在的。
  3. 任一节点是它自身的祖先（后代）。
  4. 根据上面第3点，若p已经是q的祖先了，则p是二者的LCA。

二、题解

思路

本题不同于上一题，上一题中输入的树为BST，可以通过比较节点的数值，每遍历一个节点，将问题规模缩小到原来的一半。

Solution-1：Recursive Solution

1.如果root为NULL，则return NULL

2.如果p,q中一者为root，则它俩的LCA一定为root

3.否则的话，说明：p和q一定是root的某一个子节点，分别在root的左子树和右子树中查找p,q的LCA
a.若在某个子树中找不到p和q，则代表p和q同时在本子树的兄弟子树中，此时返回NULL
b.若在某个子树中找到了p（或者q），则函数立马返回p（或者q）。
相当于说：某个后代M发现了两个目标（p或q）中的其中一个（例如p），立马报告自己的父亲A。
由于A还有一个儿子N，当M和N都给A汇报了结果后，A再进行决策。
a.若此时N也发现了目标中的另外一个，则A就是p和q的LCA。
b.若此时N没有发现目标，则A上报他自己的父亲B：p是属于以A为根的子树中的某个节点。
c.B收到这个信息，同时再看B的另外一个儿子的上报信息，决策并往上上报信息，直至root节点。
d.假设B在root的左子树中，root一看，发现p在其左子树中。同时root收到了其右子树的信息。
假设右子树中发现了q，此时root就可以说自己是p和q的LCA
假设右子树中发现了p（不可能的情况，p已在root的左子树中被发现了）
假设右子树中没有发现p,q的任何一个，即返回值为NULL。此时root一想，p已被发现，而q没有，那么p一定就是二者的LCA，则p为答案。
因为我们在发现了二者中的任意一个的时候，都立马返回，上报给父亲。而访问的顺序是dfs，说明未被发现的q一定为p的某个后代，故此p就是答案。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277036953/)

Solution-2：Iterative Solution

1.思路2：
找出从root到p的路径，存放在vectorP中：root, a, b, c, …, p
找出从root到q的路径，存放在vectorP中：root, a, b, c, …, q
从前往后对比两个vector，最后一个匹配的节点，即是答案。

2.在找从root到p、q的路径时，用dfs(preorder)的顺序：
遍历到某个节点，将其加入路径中。
如果这个节点为p或者q其中一个，则一条路径寻找完成。直到两条路径都找到，此时结束遍历。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/277045028/)
