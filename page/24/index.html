【DP专项】070.Climbing Stairs

发表于 2020-01-01 更新于 2020-01-02

一、题意

题目链接: 070.Climbing Stairs

1.Description

【输入】
给定一个正整数n，代表有n个台阶。
【输出】
假设你每次只能走一步或者两步。
求你达到楼顶，有多少种不同的方式？

2.Example

Input: 3
Output: 3
Explanation: There are three ways to climb to the top.

  1. 1 step + 1 step + 1 step
  2. 1 step + 2 steps
  3. 2 steps + 1 step

3.Corner Case

  1. n为1时，答案为1
  2. n为2时，答案为2
  3. 题目保证n是正整数

二、题解

分析

  1. 状态定义：dp[i]：有多少种不同的方法到达第i个台阶。
  2. 转移方程：dp[i] = dp[i - 2] + dp[i - 1]；答案就是dp[n - 1]；
  3. 边界条件：dp[0] = 1, dp[1] = 2
  4. 计算顺序：从2开始，到n - 1结束。

Solution-1：Dynamic Programming(Iterative)

1.dp[i] = dp[i - 2] + dp[i - 1]

2.Code(https://leetcode.com/submissions/detail/261623472/)
时间复杂度：O(n)
空间复杂度：O(n)

Solution-2：Dynamic Programming(Iterative) + Space Optimization

1.dp[i] = dp[i - 2] + dp[i - 1]

2.Code(https://leetcode.com/submissions/detail/261625030/)
时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】024.Swap Nodes in Pairs

发表于 2019-12-30

一、题意

题目链接: 024.Swap Nodes in Pairs

1.Description

【输入】
给定一个单链表。
【输出】
要求交换两两结点的顺序。如果仅有单节点，则无需处理。

2.Example

Given 1->2->3->4, you should return the list as 2->1->4->3.

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. 链表为单节点时，无需处理。

二、题解

Solution-1：Iterative Solution

1.回忆【206.Reverse Linked List】的Solution-3，对于单链表[1->2->3->NULL]：
通过一次循环，将其转变为[2->1->3->NULL]。

2.用相似的思路进行处理：
使用dummy连接在head的前面。
pre指针初始化指向dummy。含义为：待swap的两个结点的前驱结点。

3.当pre以及pre->next->next均不为NULL的情况下，进行两两一对处理：
cur为两两一对的第一个结点。
next为两两一对的第二个结点。
swap后，pre向后移动两步。
最终返回dummy.next即可。

时间复杂度：O(n)
空间复杂度：O(1)

4.Code(https://leetcode.com/submissions/detail/289732263/)

Solution-2：Recursive Solution

1.使用递归来解决：
边界条件：当子链表为单节点或者NULL时，不处理。

2.对于当前链表，头为head，过程如下：
首先求得子链表返回的结果：swapPairs(head->next->next); 将其接在head的后面。
再将head接在temp（原head的后继）的后面。

时间复杂度：O(n)
空间复杂度：O(n)

【BST专项】109.Convert Sorted List to Binary Search Tree

发表于 2019-12-30

一、题意

题目链接: 109.Convert Sorted List to Binary Search Tree

1.Description

【输入】
给定一个单链表，元素是升序排列的。
【输出】
将这个链表转化为一个高度平衡的（height-balanced）二叉搜索树（BST）。

2.Example

Input: [-10,-3,0,5,9]
Output: [0,-3,9,-10,NULL,5,NULL] 答案不唯一。

3.Corner Case

  1. 链表为NULL时，树为NULL。
  2. 链表为单节点时，树为单节点。

二、题解

Solution-1：Recursive Solution

1.给定一个单链表，要将其转化为高度平衡的BST。
a.取数组中间元素作为root；（快慢指针法）
b.递归将其右边所有元素构成一棵较小的高度平衡的BST。作为root的右子树。
c.递归将其左边所有元素构成一棵较小的高度平衡的BST。作为root的左子树。

2.结束条件：若链表为单节点，则将其作为一个单节点；若链表为NULL，则树也为NULL；

3.不管n是奇数还是偶数，对于任一节点，其左右子树的节点数不会大于1。保证了height-balanced。

4.举个例子：
n = 2时，链表为：1->2->NULL
经过快慢指针法：
slow指向1，fast指向NULL。
root为2，右子树为[NULL]，左子树为[1]。

n = 3时，链表为：1->2->3->NULL
slow指向1，fast指向3。
root为2，右子树为[3,NULL]，左子树为[1,NULL]。

n = 4时，链表为：1->2->3->4->NULL
slow指向2，fast指向NULL。
root为3，右子树为[4,NULL]，左子树为[1,2,NULL]。

n = 5时，链表为：1->2->3->4->5->NULL
slow指向2，fast指向5。
root为3，右子树为[4,5,NULL]，左子树为[1,2,NULL]。

时间复杂度：O(n)
空间复杂度：O(n)

5.Code(https://leetcode.com/submissions/detail/277846719/)
