【链表专项】141.Linked List Cycle

发表于 2019-12-29

一、题意

题目链接: 141.Linked List Cycle

1.Description

【输入】
给定一个单链表，头节点为head。
【输出】
判断上述链表是否有环，有返回true，无返回false。

2.Example

Input: head = [3,2,0,-4], pos = 1
Output: true
Explanation: There is a cycle in the linked list, where tail connects to the second node.

3.Corner Case

  1. 如果链表为NULL，则false。
  2. 如果链表为单节点，则false。
  3. 要求O(1)的space complexity。

二、题解

Solution-1：Fast and Slow Pointers

1.要想形成环，则链表中至少需要两个节点。

2.使用快慢指针法：两指针同时从head出发，slow每次走一步，fast每次走两步。
如果某个时间，它们相遇了，则说明链表中一定有环。

3.当fast以及fast->next都不为NULL的时候，fast才能向后移动到下一个位置。
否则会发生访问地址错误。

时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】083.Remove Duplicates from Sorted List

发表于 2019-12-29

一、题意

题目链接: 083.Remove Duplicates from Sorted List

1.Description

【输入】
给定一个单链表，头为head。该单链表为有序的（升序）。
【输出】
将上述单链表中value重复的元素都去除，使相同value的节点仅出现一次。

2.Example

Input: 1->1->2->3->3
Output: 1->2->3

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. 链表为单节点时，无需处理。

二、题解

Solution-1：Iterative Solution

1.指针cur初始化为head。
对于每个cur：如果发现其后续节点的value值和cur的value值相等的话，将其后继节点逐一删除。
直到其后继节点为NULL或者不为NULL，但value已经不同了。
此时cur向后移动即可。

2.直到cur为NULL，代表链表中所有的node都被考察完成了。此时return head即可。

时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】021.Merge Two Sorted Lists

发表于 2019-12-29

一、题意

题目链接: 021.Merge Two Sorted Lists

1.Description

【输入】
给定两个单链表l1、l2。它们都是有序的（升序）。
【输出】
将其合并，作为一个新的有序链表返回。

2.Example

Input: 1->2->4, 1->3->4
Output: 1->1->2->3->4->4

3.Corner Case

  1. 若l1或者l2为NULL，则新链表就是其中那个非空的，直接返回即可。

二、题解

Solution-1：Iterative Solution

1.当l1和l2同时不为空的情况下，每次考虑其中一个较小的node。
然后l1或者l2向后移动，直到两者其一为NULL。

2.cur代表当前已经merge好的链表的尾部，所以每次考察一个节点，将其接在cur的尾部即可。
然后cur向后移动。

3.使用dummyHead，来方便返回最终的链表头部。
初始化cur = &dummy;

时间复杂度：O(n)
空间复杂度：O(1)
