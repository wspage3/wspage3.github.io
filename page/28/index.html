【二叉树专项】129.Sum Root to Leaf Numbers

发表于 2019-12-25

一、题意

题目链接: 129.Sum Root to Leaf Numbers

1.Description

【输入】
给定一棵二叉树，所有节点的值均在[0, 9]中间。
每条从root到leaf的路径都可以代表一个整数。
【输出】
求出所有从root到leaf的路径，代表的整数的和。

2.Example

Input: [1,2,3]
Output: 25

3.Corner Case

  1. 树为NULL时，sum为0
  2. 树为单节点时，sum为node->val

二、题解

Solution-1：Recursive Solution(DFS)

1.遍历节点顺序为DFS(Postorder)

2.对于任一节点：
判断自身：如果自身为NULL，直接返回0；如果自身为叶子，则值 = cur->val + curSum * 10，上报给父亲。
询问儿子：将自身的值(cur->val + curSum * 10)，传递给儿子，要求它们计算。
计算自身：左儿子的值 + 右儿子的值
上报父亲：将自身的值上报给父亲

时间复杂度：O(n)
空间复杂度：O(n)

Solution-2：Iterative Solution using Stack(DFS)

1.遍历节点顺序为DFS(Preorder)

时间复杂度：O(n)
空间复杂度：O(n)

【二叉树专项】114.Flatten Binary Tree to Linked List

发表于 2019-12-25

一、题意

题目链接: 114.Flatten Binary Tree to Linked List

1.Description

【输入】
给定一棵二叉树。
【输出】
使其扁平化，退化成一个单链表。
要求【in-place】操作：此处的in-place，应该指的是在当前的节点上改变指向。而非严格O(1)的空间复杂度。
单链表的顺序为：preorder遍历树的顺序。

2.Example

Input: [1,2,5,3,4,NULL,6]
Output: [1,2,3,4,5,6]

3.Corner Case

  1. 树为NULL时，无需任何操作，return即可。

二、题解

分析

目的：使得树退化为单链表，同时单链表从前到后的顺序为：前序遍历该树的顺序。即：根 -> 左 -> 右
方法：使用相反的顺序遍历树（右 -> 左 -> 根）：每遍历一个节点，将其链接到已退化好的单链表中，更新单链表的头。

Solution-1：Recursive Solution(DFS-Postorder Traversal)

1.回忆【145.Binary Tree Postorder Traversal】的迭代版本：
我们要想得到：左 -> 右 -> 根这样的后序遍历序列。
采用的遍历顺序实际上是：根 -> 右 -> 左。最后将遍历得到的序列进行反转。

2.在本题中，采用类似的思路：
对于任一节点：
首先访问其右儿子，然后左儿子，最后访问自己。可以得到：右 -> 左 -> 根这样的后序遍历序列。
访问每个节点：将其右儿子赋值为单链表的头。将其左儿子赋值为空。将单链表的头设置为当前节点。

3.这样的访问顺序，使得最后一个访问的节点为root。
访问完root，root指针就指向一个单链表了。

时间复杂度：O(n)
空间复杂度：O(n)

4.Code(https://leetcode.com/submissions/detail/278369749/)

Solution-2：Iterative Solution

1.对于每一个节点cur，其左子树cur->left，其右子树cur->right。
考虑cur->left不为空的情况：
想办法将这两个儿子合并，成为一棵树。
这样的话，对于cur来说，就满足了链表节点的性质：只有一个后继节点。

2.由于答案顺序为：根 -> 左 -> 右
所以将右子树(cur->right)拼接到左子树(cur->left)的“最后一个”节点即可。
对于cur->left来说，“最后一个”节点即：cur->left的第一个为空的右儿子。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/278375122/)

【二叉树专项】110.Balanced Binary Tree

发表于 2019-12-25

一、题意

题目链接: 110.Balanced Binary Tree

1.Description

【输入】
给定一棵二叉树。
【输出】
判断其是否是一棵“高度平衡的二叉树”
“高度平衡的二叉树”：树中所有的节点，其左子树和右子树的高度差不大于1。

2.Example

Input: [3,9,20,null,null,15,7]
Output: true

Input: [1,2,2,3,3,null,null,4,4]
Output: false

3.Corner Case

  1. 树为NULL时，return true
  2. 树为单节点时，return true

二、题解

Solution-1：Top-to-Bottom(Dual Recursive)

1.自顶向下，分别判断每个节点的左右子树的高度，以及该左右子树是否为balanced tree。

2.缺点：双递归的存在，使得时间效率不高。计算高度存在大量重复计算。

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/276397293/)

Solution-2：Bottom-to-Top(Single Recursive)

1.自底向上，采用dfs的遍历路径（Postorder）。
对于遍历到的每一个节点，需要做以下事情：
a.询问我的左儿子是否为balanced tree，若是的话让他返回其高度，不是的话返回-1
b.询问我的右儿子是否为balanced tree，若是的话让他返回其高度，不是的话返回-1
c.现在我知道自己的两个儿子的情况了，我可以判断出自己是否为balanced tree。我判断出来，并将其告诉我的父亲

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276435936/)

【二叉树专项】199.Binary Tree Right Side View

发表于 2019-12-25

一、题意

题目链接: 199.Binary Tree Right Side View

1.Description

【输入】
给定一棵二叉树，想象你站在它的右侧。
【输出】
输出你能看到的数字，从树的top到bottom，结果存在一维数组中。

2.Example

Input: [1,2,3,null,5,null,4]
Output: [1, 3, 4]

3.Corner Case

  1. 树为NULL时，返回空数组。

二、题解

分析

本题实质就是求：树的每一层中，最右边的那个元素。如果树有k层，则答案中就包含k个数据。

Solution-1：Recursive Solution(DFS)

1.回想【102.Binary Tree Level Order Traversal】，我们同样也是采用recursive的dfs来遍历树：
如果是第一次进入该level，则新建一个sub数组存储在该level上的节点。遍历顺序：根 -> 左儿子 -> 右儿子

2.在本题中，采用根 -> 右儿子 -> 左儿子的遍历顺序。使得：第一次进入某level的就是最右边的那个节点。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276672883/)

Solution-2：Iterative Solution using Queue

1.回想【102.Binary Tree Level Order Traversal】的迭代版本，我们使用了队列：
每一轮，放入同一层中的所有节点，将其统一加入一个sub一维数组中，再将这个一维数组加入到二维答案中。

2.本题中，我们同样采用这样的遍历方法，当每一轮中访问同一层的所有节点时，我们将最后的那个节点放入ans中即可。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276668971/)

【BST专项】108.Convert Sorted Array to Binary Search Tree

发表于 2019-12-25 更新于 2019-12-27

一、题意

题目链接: 108.Convert Sorted Array to Binary Search Tree

1.Description

【输入】
给定一个长度为n的数组，数组是升序排列的。
【输出】
将这个数组转化为一个高度平衡的（height-balanced）二叉搜索树（BST）。

2.Example

Input: [-10,-3,0,5,9]
Output: [0,-3,9,-10,NULL,5,NULL] 答案不唯一。

3.Corner Case

  1. n = 0时，树为NULL。
  2. n = 1时，树为单节点。

二、题解

Solution-1：Recursive Solution

1.给定一个长度为n的数组nums，要将其转化为高度平衡的BST。
a.取数组中间元素作为root；
b.递归将其左边所有元素构成一棵较小的高度平衡的BST。作为root的左子树。
c.递归将其右边所有元素构成一棵较小的高度平衡的BST。作为root的右子树。

2.结束条件：若数组长度为1，则将其作为一个单节点。

3.不管n是奇数还是偶数，对于任一节点，其左右子树的节点数不会大于1。保证了height-balanced。

时间复杂度：O(n)
空间复杂度：O(n)

4.Code(https://leetcode.com/submissions/detail/277835456/)

Solution-2：Recursive Solution with Optimization

1.优化：递归调用的时候，避免：构建子数组，将子数组作为参数传递。
而是直接传递索引即可。

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/288910868/)
