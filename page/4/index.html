Leetcode739-Daily Temperatures

一、题意

题目链接: Leetcode739-Daily Temperatures

1.Description

给定一个长度为n的整数数组T；T[i]代表第i天的气温；
求出一个长度为n的整数数组res；res[i]代表第i天需要等待的天数，直到后面某一天气温比当天高/

2.Example

input = [73, 74, 75, 71, 69, 72, 76, 73]
output = [1, 1, 4, 2, 1, 1, 0, 0]

3.Corner Case

n：[1, 30000]
T[i]：[30, 100]

二、题解

对于当前位置的元素T[i]，找出它后面一个比他大的元素，使得他们俩的距离最小。

Solution-1：Brute Force

1.i从[0, n - 1]进行遍历
对于T[i]来说，j考虑[i + 1, n - 1]内所有的数，如果发现第一个比T[i]大的数字，则：res[i] = (j - i)

时间复杂度：O(n ^ 2)
空间复杂度：O(1)：不包含函数返回的数组res
结果发现：此方法TLE，说明需要进一步优化。

2.Code(https://leetcode.com/submissions/detail/283812278/)

Solution-2：Monotonous Stack

1.认识单调栈
https://www.cnblogs.com/grandyang/p/8887985.html
单调栈的一大优势就是线性的时间复杂度，所有的元素只会进栈一次，而且一旦出栈后就不会再进来了。
单调递增栈可以找到左边第一个比当前数字小的元素。（本题做法）
单调递减栈可以找到右边第一个比当前数字大的元素。

2.[73, 74, 71, 60, 50, 55]
先将73压入栈

对于74，此时栈非空，则考察栈顶元素73。
此时74 > 73，说明：对于73来说，找到了第一个比自己大的元素。
此时73的答案已经找到，将其从栈中拿出。
将74压入栈。

对于71，此时栈非空，则考察栈顶元素74。
此时71 < 74，说明：对于71来说，还是没有找到第一个比自己大的元素。
将71入栈。

对于60，50也是如此。此时栈由顶到下为：50，60，71，74

对于55来说，此时栈顶是50
此时55 > 50，说明：对于50来说，找到了答案。
将其出栈。继续考察新的栈顶元素60，不满足了。
将55入栈。
此时栈由顶到下为： 55，60，71，74。而此时已经遍历完所有的数字了，说明这些元素找不到比自己大的元素了。

3.可以发现，栈中数据在任何时刻都是有序的，且：最上面的数字最小，最下面的数字最大。
考虑遍历到位置nums[i]的时候，此时栈中元素自顶向下为:[a, b, c, d]。
若nums[i]不大于a，说明：nums[i]的出现，不会使得栈中任意元素找到答案。此时将nums[i]压入栈即可。
若nums[i]大于d，说明：nums[i]的出现，使得栈中任意元素都找到了答案。此时从栈顶依次拿出一个元素，直到栈空了。然后nums[i]入栈。
若nums[i]介于这两者中间的话，则“尽自己最大的能力，使得栈中的元素找到答案，然后将其出栈”
经过这样遍历的顺序：使得栈中任意时刻的数都是有序的，也使得了每个元素找到的答案都是距离自己最小的那个，也就是题目中要求的这样。

时间复杂度：O(n) 因为所有元素只会进入栈一次，并且出栈后再也不会进栈了。
空间复杂度：O(n)

Leetcode300-Longest Increasing Subsequence

一、题意

题目链接: Leetcode300-Longest Increasing Subsequence

1.Description

给定一个长度为n的整数数组nums；数组无序；
求出一个最大序列的长度。该序列内所有元素是递增的。序列不同于子数组，不要求连续。

2.Example

Input: [10,9,2,5,3,7,101,18]
Output: 4
Explanation: The longest increasing subsequence is [2,3,7,101], therefore the length is 4.

3.Corner Case

若n为0/1，直接返回0/1即可。

二、题解

【LIS】相关的题目。
序列不要求连续，所以相较于【最长递增子数组】来说，范围更大。

Solution-1：Dynamic Programming

1.考虑以下数组[a, b, c, d, e]
设res[i]表示以nums[i]结尾的递增子序列的最大长度。那题目要求的答案便是：res数组的最大值了。
显然，res[0] = 1；res[1]有两种情况：
若nums[1] <= nums[0]，则说明以nums[1]结尾的递增子序列只有一种：就是nums[1]本身。
若nums[1] > nums[0]，则说明以nums[1]可以和nums[0]连接起来，形成一个更长的子序列。
……
考虑nums[i]：
若nums[i]都不大于nums[0, i - 1]的任一数字，那res[i] = 1;
若能在[0, i - 1]区间找到k个比nums[i]小的值（k >= 1），则找出这k次“拼接”后的最大值。

2.对于每一个res[i]，
“拼接情况”：需要遍历[0, i - 1]的元素，进行判断和统计最大值maxLinkI。
“无法拼接情况”：1
res[i] = max(1, maxLinkI);

时间复杂度：O(n ^ 2)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/283644219/)

Solution-2：Dynamic Programming + Search

1.现象1：若同时有2个子序列(1, 5)和(1, 10)，然后取一个随机数x，能构成长度为3的递增子序列吗？
答案肯定是可以的，例如：
x < 1 :都ok
x > 10 :都ok
x == 2/3/4 :都不行
x == 6/7/8/9 :(1, 5)来说ok，(1, 10)来说不行
看起来对于(1, 5)和(1, 10)来说，能扩展成长度为3的递增子序列的“成功率”是不相等的，且(1, 5)的“扩展性”更好。

2.令tail[i]表示：长度为i的满足条件的若干个序列中，序列末尾元素的最小值。
tail[1]初始为nums[0]；此时满足条件的子序列的最大长度就是1；
对于nums[i]，此时满足条件的子序列的最大长度是maxLen。
如果nums[i] > tail[maxLen]的话，说明可以在当前maxLen的基础上，添加nums[i]形成一个maxLen + 1的子序列。
如果nums[i] < tail[1]的话，说明长度为1的子序列的最小值就是nums[i]，此时更新tail[1]。
如果 tail[1] == nums[i] || nums[i] == tail[maxLen]的话，不做任何操作
如果 tail[1] < nums[i] < tail[maxLen]的话，说明可以更新某个tail[j]的值，来使得长度为j的子序列的“扩展性”更好。
所以每次我们在[1, maxLen]上从后往前查找，如果发现nums[i]大于了tail[j]，就使得：tail[j + 1] = nums[i]

时间复杂度：O(n ^ 2)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/283675471/)

Solution-2：Dynamic Programming + Binary Search

1.不难发现，tail数组是递增的，因此，我们可以用二分搜索进一步降低时间复杂度。

时间复杂度：O(n ^ logn)
空间复杂度：O(n)

Leetcode713-Subarray Product Less Than K

一、题意

题目链接: Leetcode713-Subarray Product Less Than K

1.Description

给定一个长度为n的正整数数组nums；给定一个整数k；
求出满足条件的连续子数组的个数cnt：这个子数组的product小于给定的k

2.Example

Input: nums = [10, 5, 2, 6], k = 100
Output: 8
Explanation:
[10], [5], [2], [6], [10, 5], [5, 2], [2, 6], [5, 2, 6]

3.Corner Case

0 < nums.length <= 50000.
0 < nums[i] < 1000.
0 <= k < 10^6.
需要对k是否为0判断：若k为0，根据题目正整数数组的描述，直接返回0即可

二、题解

【连续子数组的乘积】相关的题目。

Solution-1：Brute Force

1.i从[0, n - 1]遍历。
每次固定i，然后j从[i, n - 1]遍历：找出以i开头所有的连续子数组。其中累乘product，若小于k，则计数器加一。
结果发现Runtime Error：累乘的数据太大了，以至于用long long也会溢出。
2.一个显而易见有问题的地方在于：
给定的k的范围为[0, 1e6]，而我们固定i，试图找出以i开头的所有连续子数组的时候，为什么还会导致累乘的积溢出long long呢？
显然是因为：当发现累乘的积大于k的时候，没有跳出循环，直接考虑i + 1。（由于nums数组中的元素都是正数）
修改后发现结果是：TLE超时。

时间复杂度：O(n ^ 2)
空间复杂度：O(1)

2.Code(https://leetcode.com/submissions/detail/283569919/)

Solution-2：Sliding Window

考虑数组[a, b, c, d, e, f, g, h, i]

1.首先我们考虑以a开头的子树组：a,ab,abc,abcd,abcde……
若a本身大于等于k，则以a开头的所有子数组都不符合条件，跳过到b。
若a本身不大于k，则可以继续考察ab,abc,abcd,abcde……，某个时刻发现abcdefg不符合条件了：
但我们知道：abcdef是ok的，说明：abcdef内的任意连续子数组都是满足的。
Solution1的做法是break后考察以b开头的。这样的话，就会有重复计算了，不能充分利用上述已知信息。

2.退一步，假设当前位置在”f”，我们已经知道了abcde是合法的，同时abcdef也是合法的。
考虑：把当前的”f”添加到abcde中，新增了一个元素，对应新增了多少个满足条件的子数组呢？
结果是：f, ef, def, cdef, bcdef, abcdef共6个，也就是index(f) - index(a) + 1。
同理我们知道序列abcde对应的答案为：1 + 2 + 3 + 4 + 5 = 15
那么abcdef对应的答案就是：15 + 6 = 21

3.进一步，当前位置在”g”，当发现abcdefg不合法的时候，说明此时答案无法在21的基础上直接加上7。
这时，我们找到以g为结尾的满足条件的一个最大子数组，假设是序列cdefg。即：abcdefg,bcdefg均不合法；而cdefg合法。
此时就可以将g对应的cnt添加到已有的21中了：g, fg, efg, defg, cdefg。
而且这些包含g的子数组不可能和之前的21个子数组有相同的。
同时我们跳过的a和b，他们所有满足的情况都已经被考虑过了。也不会造成遗漏。
然后我们再在这个cdefg的基础上，考察下一个位置”h”……直到所有位置都被考察过了，最后的答案就是本题目的答案。

4.根据以上思路，进行实现：
用一个start指针指向当前window的开始，初始化为0。
每次考察数组nums[i]，i的范围[0, n - 1]，依次将nums[i]添加到window中，并进行判断。
如果此时合法，则考察[i + 1]。并累加：i - start + 1。
如果此时非法，则将window向后滑动，找到start的位置，使得区间[start, i](start <= i>)合法。并累加：i - start + 1。
如果此时发现start > i，则说明start == i + 1，且i位置不存在满足条件的子数组，累加0即可。然后考察i + 1

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/283582840/)

Leetcode-152 Maximum Product Subarray

一、题意

题目链接: Leetcode-152 Maximum Product Subarray

1.Description

给定一个长度为n的整数数组nums。
求出nums的连续子数组，这个子数组是nums所有子数组中乘积最大的那个。返回这个sum。
子数组长度至少为1。

2.Example

Input: [2,3,-2,4]
Output: 6
Explanation: [2,3] has the largest product 6.

Input: [-2,0,-1]
Output: 0
Explanation: The result cannot be 2, because [-2,-1] is not a subarray.

3.Corner Case

需要对n是否为0判断

二、题解

显然，本题目和Leetcode-53的唯一区别就是：一个求和最大值，一个求乘积最大值。
此处，主要需要考虑乘法运算和加法元素的差异。其它分析可参考Leetcode-53

Solution-1：Brute Force

1.不感知加法乘法的差异。
时间复杂度：O(n ^ 2)
空间复杂度：O(1)

2.Code(https://leetcode.com/submissions/detail/283351227/)

Solution-2：Divide and Conquer

1.考虑区间[low, high]，mid为中点：
递归求得[low, mid]和[mid + 1, high]的值。
找出[i, mid]使得以mid结尾的最大值。
找出[mid + 1, j]使得以mid + 1开头的最大值。
假设区间为：[-3, 2, -5]
f(-3, 2) = 2
f(-5) = -5
f(i, j) = 2 * -5 = -10
这样求得的值是2，跟实际答案30不符合。
可以发现，问题出现在“负负得正”这种场景下。我们在寻找[i, mid]和[mid + 1, j]的时候，单纯的找到了以其开头或者结尾的最大的值。但这两个最大的值的乘积并不一定是最大的。
所以根据这个特点，可以分别维护两个值。
对于i来说，lMax，lMin分别记录以mid结尾的最大乘积和最小乘积。
对于j来说，rMax，rMin分别记录以mid + 1开头的最大乘积和最小乘积。
当combine的时候，找出max(lMax * rMax, lMin * rMin)，这样才是[i, j]范围内的最大乘积。

时间复杂度：O(n ^ logN)
空间复杂度：O(n)，递归开销
3.Code(https://leetcode.com/submissions/detail/283358210/)

Solution-3：Dynamic Programming

1.同理，在递推位置i的时候：
sum：
start = max(nums[i], nums[i] + start)
res = max(res, start)
product：
if (nums[i] < 0) then swap(startMin, startMax);
startMin = min(nums[i], nums[i] * startMin);
startMax = max(nums[i], nums[i] * startMax);
res = max(res, startMax);
当在位置i的时候，此时：
startMin代表以i + 1开头的最小乘积
startMax代表以i + 1开头的最大乘积
如果当前nums[i]非负的话，要想得到以i开头的连续子树组的最大值，并更新它的话，需要结合maxStart。
maxStart = max(nums[i], nums[i] * maxStart)
同理更新minStart：minStart = max(nums[i], nums[i] * minStart)
如果当前nums[i]负数的话，要想得到以i开头的连续子树组的最大值，并更新它的话，需要结合minStart。
maxStart = max(nums[i], nums[i] * minStart)
同理更新minStart：minStart = max(nums[i], nums[i] * maxStart)
为了表达简便，我们可以在更新前判断，如果当前nums[i]是负数的话，将minStart和maxStart交换值即可。

时间复杂度：O(n)
空间复杂度：O(1)

Leetcode-53 Maximum Subarray

一、题意

题目链接: Leetcode-53 Maximum Subarray

1.Description

给定一个长度为n的整数数组nums。
求出nums的连续子数组，这个子数组是nums所有子数组中和最大的那个。返回这个sum。
子数组长度至少为1。

2.Example

Input: [-2,1,-3,4,-1,2,1,-5,4],
Output: 6
Explanation: [4,-1,2,1] has the largest sum = 6.

3.Note

a.多种方法求解，使用一下Divide and Conquer方法
b.某种方法的时间复杂度为O(n)

4.Corner Case

需要对n是否为0判断

二、题解

Solution-1：Brute Force

1.直观的想法就是遍历n次，每次选定一个nums[i]；i的范围[0, n - 1]。
然后对于该nums[i]，考察以它为首的所有子数组的和，并更新这个最大值；j的范围[i, n - 1]。
这样的话，可以将nums所有可能的子数组都遍历一次，不会遗漏。
时间复杂度：O(n ^ 2)
空间复杂度：O(1)

2.Code(https://leetcode.com/submissions/detail/283334388/)

Solution-2：Divide and Conquer

1.现有数组[a, b, c, d, e]，把它分为左右两部分：[a, b, c]和[d, e]。
若已经知道左右两部分的答案分别为A和B，能否求出本题的答案C？
考虑以下三种情况：
a.若答案C对应的子数组就在[a, b, c]范围内，则在[a, b, c, d, e]中的答案等于在[a, b, c]中的答案。如：[1,2,3,-1,-1]
b.若答案C对应的子数组就在[d, e]范围内，则在[a, b, c, d, e]中的答案等于在[d, e]中的答案。如：[-2,-3,-5,7,7]
c.若答案C对应的子数组在穿过了左右两部分，如[1,2,3,4,5]，则我们可以知道：这个子数组中必然包含[c,d]两个元素。
也就是说，只要以c结尾的连续子数组中取最大值left，以d开头的连续子数组中取最大值right；left + right就是这种情况下的答案。
可以发现，上述三种情况覆盖了所有可能出现的情况，只要我们找出这三个值的最大值，也就是最后要求的答案了。

2.对于一个区间[low, high]，求这个范围内的答案，只要把上面三种情况下的最大值找出来，就是答案。
如果区间中有1个元素[a]，那么答案就是a本身；
记为f(a) = a；
如果区间中有2个元素[a, b]，那么答案就是max(a, b, (a + 0) + (b + 0))；(a+0)代表以a为结尾的子数组的最大值；(b+0)同理。
记为f(a, b) = max(a, b, (a + 0) + (b + 0))；
如果区间中有3个元素[a, b, c]，那么答案就是max(f(a, b), c, max(b, a + b) + c)；max(b, a + b)代表以b为结尾的子数组的最大值，c同理。
记为f(a, b, c) = max(f(a, b), c, max(b, a + b) + c)；max(b, a + b)
所以：对于求区间[low, high]上的答案，只要分别求得左子区间[low, mid]，右子区间[mid + 1, high]，左右子区间[i, j]。
其中，左子区间[low, mid]，右子区间[mid + 1, high]可以递归求得，当low == high时，到达递归的结束条件，返回nums[low]即可。
左右子区间的i，可以由mid位置，依次递减到low，每次叠加nums中的元素，找出一个最大和。
左右子区间的j，可以由mid + 1位置，依次递增到high，每次叠加nums中的元素，找出一个最大和。

时间复杂度：O(n ^ logN)
空间复杂度：O(n)，递归开销
3.Code(https://leetcode.com/submissions/detail/283548839/)

Solution-3：Dynamic Programming

1.顺着上面的想法继续往下思考。
考虑新增两个数组res和start：
vector res(n); //res[i]表示[i, n - 1]内最大连续子数组的和。则res[0]就是我们要找的答案。显然，res[n - 1] = nums[n - 1]
vector start(n); //start[i]表示[i, n - 1]内最大连续子数组的和，且该子数组以i为开头。同理，start[n - 1] = nums[n - 1]
在元素n - 1考虑完成后，我们考虑它前面的n - 2：
start[n - 2] = max(nums[n - 2], nums[n - 2] + start[n - 1])
res[n - 2] = max(start[n - 2], res[n - 1])，其中前者代表：当前元素存在于答案中，后者代表：当前元素不存在于答案中
依次向前遍历，经过一次循环，即可找到答案，也就是res[0]。
时间复杂度：O(n)
空间复杂度：O(n * 2)

2.Code(https://leetcode.com/submissions/detail/283366882/)

Solution-4：Further Optimization

1.能否进一步优化空间复杂度呢？
当我们考虑第i个位置的时候，只需要得到nums[i]，start[i + 1]，res[i + 1]即可求出。
也就是说，第i个位置的答案只与它后边的一个元素有关，和其他的都无关。所以可以用两个int变量res和start分别替换上述的两个int数组。
时间复杂度：O(n)
空间复杂度：O(1)

Leetcode-238 Product of Array Except Self

一、题意

题目链接: Leetcode-238 Product of Array Except Self

1.Description

给定一个长度为n的整数数组nums，n > 1。
求数组output，其中output[i]表示所有除了i之外，所有其它数字的乘积。

2.Example

Input: [1,2,3,4]
Output: [24,12,8,6]

3.Note

a.过程中不使用除法。
b.时间复杂度要求：O(n)。
c.空间复杂度要求：O(1)，函数返回的数组不算入空间复杂度。

4.Corner Case

题目保证了n > 1

二、题解

Solution-1：Brute Force

直观的想法就是遍历n次，每次累乘(n - 1)个数字。
时间复杂度：O(n ^ 2)
空间复杂度：O(1)

Solution-2：递推

1.假设我们要求output[i]，i的范围为：[0, n - 1]，也就是nums数组中除了nums[i]之外所有的数字的乘积。也就是求[0, i - 1]以及[i + 1, n - 1]两个范围内所有数的乘积。

2.新增数组s，设数组s[i]表示范围[0, i - 1]内所有数字的乘积。i的范围为：[1, n - 1]
新增数组t，设t[i]为[i + 1, n - 1]所有数字的乘积。i的范围为：[0, n - 2]
数组s只需要从前到后遍历一次nums即可求出。
数组t从后到前遍历一次nums也能求出。
在s和t都求出的情况下，output也只需要O(n)的时间即可求出：output[i] = s[i] * t[i];
时间复杂度：O(3 * n)
空间复杂度：O(2 * n)

3.Code(https://leetcode.com/submissions/detail/283206956/)

Solution-3：进一步优化

1.可以省略数组s，和数组t：
数组s的值就保留在output数组中。
数组t用一个int变量表示。
时间复杂度：O(2 * n)
空间复杂度：O(1)
