【链表专项】024.Swap Nodes in Pairs

发表于 2019-12-30

一、题意

题目链接: 024.Swap Nodes in Pairs

1.Description

【输入】
给定一个单链表。
【输出】
要求交换两两结点的顺序。如果仅有单节点，则无需处理。

2.Example

Given 1->2->3->4, you should return the list as 2->1->4->3.

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. 链表为单节点时，无需处理。

二、题解

Solution-1：Iterative Solution

1.回忆【206.Reverse Linked List】的Solution-3，对于单链表[1->2->3->NULL]：
通过一次循环，将其转变为[2->1->3->NULL]。

2.用相似的思路进行处理：
使用dummy连接在head的前面。
pre指针初始化指向dummy。含义为：待swap的两个结点的前驱结点。

3.当pre以及pre->next->next均不为NULL的情况下，进行两两一对处理：
cur为两两一对的第一个结点。
next为两两一对的第二个结点。
swap后，pre向后移动两步。
最终返回dummy.next即可。

时间复杂度：O(n)
空间复杂度：O(1)

4.Code(https://leetcode.com/submissions/detail/289732263/)

Solution-2：Recursive Solution

1.使用递归来解决：
边界条件：当子链表为单节点或者NULL时，不处理。

2.对于当前链表，头为head，过程如下：
首先求得子链表返回的结果：swapPairs(head->next->next); 将其接在head的后面。
再将head接在temp（原head的后继）的后面。

时间复杂度：O(n)
空间复杂度：O(n)

【链表专项】019.Remove Nth Node From End of List

发表于 2019-12-30

一、题意

题目链接: 019.Remove Nth Node From End of List

1.Description

【输入】
给定一个单链表，头为head。
给定一个正整数n：[0, num]。其中num是链表中结点的数目。
【输出】
将上述单链表中从后面数第n个结点删除掉。
注意：编号不管从前还是从后，都是从1开始。

2.Example

Given linked list: 1->2->3->4->5, and n = 2.
After removing the second node from the end, the linked list becomes 1->2->3->5.

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. n为0时，无需处理。
  3. one pass处理。即：仅仅遍历链表一次。

二、题解

Solution-1：Iterative Solution

1.常规思路：先获取到链表中有多少结点，然后推算出来从前往后是要删除哪一个节点。
再经过一次遍历，删除对应的结点。

2.本题要求one pass处理，即不允许遍历链表两次。可以使用快慢指针法：
slow指针初始化为head
fast指针初始化为head，fast先向后走n步（1 <= n <= num）。此时检查fast是否为NULL。
a.如果NULL，说明n == num，要删除链表中第一个元素。
b.如果不为NULL的话，说明n < num，要删除的结点非head。

3.此时slow，fast同时出发，它们之间相距n个结点。
它们俩同时往后移动，直到fast遍历到链表中的最后一个结点，停止。此时slow->next即要删除的结点。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/279781260/)

【链表专项】082.Remove Duplicates from Sorted List II

发表于 2019-12-30

一、题意

题目链接: 082.Remove Duplicates from Sorted List II

1.Description

【输入】
给定一个单链表，头为head。该单链表为有序的（升序）。
【输出】
将上述单链表中value出现多次的元素都去除（一个也不留），仅出现一次的元素保留。

2.Example

Input: 1->2->3->3->4->4->5
Output: 1->2->5

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. 链表为单节点时，无需处理。

二、题解

Solution-1：Iterative Solution

1.回忆【203.Remove Linked List Elements】中，我们遍历链表来寻找所有value为target的节点，并将其删除。
使用一个dummyHead接在链表的前面，初始化cur指向dummyHead。依次检查cur->next是否满足题意……直到cur为空。

2.回忆【083.Remove Duplicates from Sorted List】中，我们遍历链表来寻找所有value重复的节点，保留第一个，将后面的删除。

3.本题中，也使用一个dummyHead接在链表前面。因为链表中第一个元素同样可能是满足题意的，即需要删除的。
pre指针初始化指向dummy。
cur指针初始化执行head。

4.外层循环：当cur非空的时候，考虑cur。
内层循环：考虑cur紧接着的后面若干个元素，其value是否与cur->value相等。相等的话，删除cur后继的并且值相等的元素。
同时置flag为true，表示当前cur也需要删除。

时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】328.Odd Even Linked List

发表于 2019-12-30

一、题意

题目链接: 328.Odd Even Linked List

1.Description

【输入】
给定一个单链表。
【输出】
要求将该单链表中的所有奇数项节点放在所有偶数项节点前面。
同时所有奇数项或者所有偶数项的相对顺序要保持不变。
即：1，3，5，7，…，2，4，6，8，…，NULL
注：链表第一个节点序号为1。

2.Example

Input: 2->1->3->5->6->4->7->NULL
Output: 2->3->6->7->1->5->4->NULL

3.Corner Case

  1. 此处的odd、even指的是节点的序号，而非节点的value。
  2. 若链表为NULL，无需处理。
  3. 若链表中节点数目小于等于2，无需处理。
  4. 要求in-place就地处理（O(1)的space），同时O(n)的time。

二、题解

Solution-1：Iterative Solution

1.当链表中节点数目小于等于2时，无需处理。

2.定义如下：
headNext始终指向第二个元素。
headOdd指向奇数项(1.3.5…)元素，初始化指向第1个节点
headEven指向偶数项(2.4.6…)元素，初始化指向第2个节点

3.过程如下：
a.当headEven为NULL或者headEven->next为NULL的时候，代表headEven不能再向后移动了，此时循环结束。
b.否则的话：headOdd和headEven指针依次向后移动，链接对应的节点。
c.最后当循环退出的时候，将奇数项链表的尾部接在偶数项链表的头部：headOdd->next = headNext;

4.举个例子：
当n == 3的时候，链表为：1->2->3->NULL
循环结束后：
head：1->3->NULL，headOdd为3。
headNext：2->NULL，headEven为NULL。
再进行链接：headOdd->next = headNext;

当n == 4的时候，链表为：1->2->3->4->NULL
循环结束后：
head：1->3->4->NULL，headOdd为3。
headNext：2->4->NULL，headEven为4。
再进行链接：headOdd->next = headNext;

时间复杂度：O(n)
空间复杂度：O(1)

【BST专项】109.Convert Sorted List to Binary Search Tree

发表于 2019-12-30

一、题意

题目链接: 109.Convert Sorted List to Binary Search Tree

1.Description

【输入】
给定一个单链表，元素是升序排列的。
【输出】
将这个链表转化为一个高度平衡的（height-balanced）二叉搜索树（BST）。

2.Example

Input: [-10,-3,0,5,9]
Output: [0,-3,9,-10,NULL,5,NULL] 答案不唯一。

3.Corner Case

  1. 链表为NULL时，树为NULL。
  2. 链表为单节点时，树为单节点。

二、题解

Solution-1：Recursive Solution

1.给定一个单链表，要将其转化为高度平衡的BST。
a.取数组中间元素作为root；（快慢指针法）
b.递归将其右边所有元素构成一棵较小的高度平衡的BST。作为root的右子树。
c.递归将其左边所有元素构成一棵较小的高度平衡的BST。作为root的左子树。

2.结束条件：若链表为单节点，则将其作为一个单节点；若链表为NULL，则树也为NULL；

3.不管n是奇数还是偶数，对于任一节点，其左右子树的节点数不会大于1。保证了height-balanced。

4.举个例子：
n = 2时，链表为：1->2->NULL
经过快慢指针法：
slow指向1，fast指向NULL。
root为2，右子树为[NULL]，左子树为[1]。

n = 3时，链表为：1->2->3->NULL
slow指向1，fast指向3。
root为2，右子树为[3,NULL]，左子树为[1,NULL]。

n = 4时，链表为：1->2->3->4->NULL
slow指向2，fast指向NULL。
root为3，右子树为[4,NULL]，左子树为[1,2,NULL]。

n = 5时，链表为：1->2->3->4->5->NULL
slow指向2，fast指向5。
root为3，右子树为[4,5,NULL]，左子树为[1,2,NULL]。

时间复杂度：O(n)
空间复杂度：O(n)

5.Code(https://leetcode.com/submissions/detail/277846719/)
