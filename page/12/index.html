【链表专项】141.Linked List Cycle

发表于 2019-12-29

一、题意

题目链接: 141.Linked List Cycle

1.Description

【输入】
给定一个单链表，头节点为head。
【输出】
判断上述链表是否有环，有返回true，无返回false。

2.Example

Input: head = [3,2,0,-4], pos = 1
Output: true
Explanation: There is a cycle in the linked list, where tail connects to the second node.

3.Corner Case

  1. 如果链表为NULL，则false。
  2. 如果链表为单节点，则false。
  3. 要求O(1)的space complexity。

二、题解

Solution-1：Fast and Slow Pointers

1.要想形成环，则链表中至少需要两个节点。

2.使用快慢指针法：两指针同时从head出发，slow每次走一步，fast每次走两步。
如果某个时间，它们相遇了，则说明链表中一定有环。

3.当fast以及fast->next都不为NULL的时候，fast才能向后移动到下一个位置。
否则会发生访问地址错误。

时间复杂度：O(n)
空间复杂度：O(1)

【链表专项】083.Remove Duplicates from Sorted List

发表于 2019-12-29

一、题意

题目链接: 083.Remove Duplicates from Sorted List

1.Description

【输入】
给定一个单链表，头为head。该单链表为有序的（升序）。
【输出】
将上述单链表中value重复的元素都去除，使相同value的节点仅出现一次。

2.Example

Input: 1->1->2->3->3
Output: 1->2->3

3.Corner Case

  1. 链表为NULL时，无需处理。
  2. 链表为单节点时，无需处理。

二、题解

Solution-1：Iterative Solution

1.指针cur初始化为head。
对于每个cur：如果发现其后续节点的value值和cur的value值相等的话，将其后继节点逐一删除。
直到其后继节点为NULL或者不为NULL，但value已经不同了。
此时cur向后移动即可。

2.直到cur为NULL，代表链表中所有的node都被考察完成了。此时return head即可。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/280524327/)

【链表专项】021.Merge Two Sorted Lists

发表于 2019-12-29

一、题意

题目链接: 021.Merge Two Sorted Lists

1.Description

【输入】
给定两个单链表l1、l2。它们都是有序的（升序）。
【输出】
将其合并，作为一个新的有序链表返回。

2.Example

Input: 1->2->4, 1->3->4
Output: 1->1->2->3->4->4

3.Corner Case

  1. 若l1或者l2为NULL，则新链表就是其中那个非空的，直接返回即可。

二、题解

Solution-1：Iterative Solution

1.当l1和l2同时不为空的情况下，每次考虑其中一个较小的node。
然后l1或者l2向后移动，直到两者其一为NULL。

2.cur代表当前已经merge好的链表的尾部，所以每次考察一个节点，将其接在cur的尾部即可。
然后cur向后移动。

3.使用dummyHead，来方便返回最终的链表头部。
初始化cur = &dummy;

时间复杂度：O(n)
空间复杂度：O(1)

4.Code(https://leetcode.com/submissions/detail/289437390/)

【链表专项】206.Reverse Linked List

发表于 2019-12-29

一、题意

题目链接: 206.Reverse Linked List

1.Description

【输入】
给定一个单链表。
【输出】
将其反转。

2.Example

Input: 1->2->3->4->5->NULL
Output: 5->4->3->2->1->NULL

3.Corner Case

  1. 链表为NULL时，无需操作。
  2. 链表为单节点时，无需操作。
  3. 递归法 + 迭代法

二、题解

Solution-1：Iterative Solution

1.head指针从前往后遍历，直到其为NULL，代表所有的节点都被考察完成了。

2.每遍历一个节点，head指针向后移动之前，先将当前位置赋值给pre。

3.prepre指向已经逆序好的链表头。
所以，每一次，pre只要将其next指针赋值为prepre即可：代表着将当前节点挂在已有逆序链表的头。

4.最后更新prepre即可。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/279788039/)

Solution-2：Recursive Solution

1.使用递归：
对于链表head，先调用reverseList(head->next);
然后将head放在其已经逆序好的链表末尾。

2.确定结束条件：当链表为NULL或者单节点时，无需reverse。

时间复杂度：O(n)
空间复杂度：O(1)

6.Code(https://leetcode.com/submissions/detail/279789541/)

Solution-3：Another Iterative Solution

1.使用临时节点dummy，将其放在head的前面。

2.pre始终指向dummy，cur始终指向原始的head。

3.例如1->2->3->4->5->NULL，过程如下：
第一轮：2->1->3->4->5->NULL
第二轮：3->2->1->4->5->NULL
第三轮：4->3->2->1->5->NULL
第四轮：5->4->3->2->1->NULL

4.最后返回dummy.next(pre->next)即可。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/289498490/)

【链表专项】203.Remove Linked List Elements

发表于 2019-12-29

一、题意

题目链接: 203.Remove Linked List Elements

1.Description

【输入】
给定一个单链表，头为head。
给定一个整数val。
【输出】
在上述链表中，删除所有值为val的节点。

2.Example

Input: 1->2->6->3->4->5->6, val = 6
Output: 1->2->3->4->5

3.Corner Case

  1. 链表为NULL的时候，不做任何操作。
  2. 删除后，链表为空的时候，返回NULL。
  3. 删除后，链表不为空的时候，返回newHead。

二、题解

Solution-1

1.回忆【237.Delete Node in a Linked List】，给定一个目标节点node，通过将其后继节点赋值给node的办法，在链表中删除该节点node。

2.在本题中，给定了head，如果遍历到某个节点，发现其需要删除时，可以将其前一个节点和其后继节点接起来：
由于访问当前节点，需要使用到前一个节点。
所以可以对于每个节点，通过检查其next节点，如果其需要删除，则将其后继接在本节点的后面。

3.当链表头需要被删除时，可以使用dummyHead的方法：将一个临时节点dummy接在原有head的前面。最后直接返回dummy.next即可。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/280528928/)
