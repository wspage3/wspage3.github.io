【链表专项】206.Reverse Linked List

发表于 2019-12-29

一、题意

题目链接: 206.Reverse Linked List

1.Description

【输入】
给定一个单链表。
【输出】
将其反转。

2.Example

Input: 1->2->3->4->5->NULL
Output: 5->4->3->2->1->NULL

3.Corner Case

  1. 链表为NULL时，无需操作。
  2. 链表为单节点时，无需操作。
  3. 递归法 + 迭代法

二、题解

Solution-1：Iterative Solution

1.head指针从前往后遍历，直到其为NULL，代表所有的节点都被考察完成了。

2.每遍历一个节点，head指针向后移动之前，先将当前位置赋值给pre。

3.prepre指向已经逆序好的链表头。
所以，每一次，pre只要将其next指针赋值为prepre即可：代表着将当前节点挂在已有逆序链表的头。

4.最后更新prepre即可。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/279788039/)

Solution-2：Recursive Solution

1.使用递归：
对于链表head，先调用reverseList(head->next);
然后将head放在其已经逆序好的链表末尾。

2.确定结束条件：当链表为NULL或者单节点时，无需reverse。

时间复杂度：O(n)
空间复杂度：O(1)

6.Code(https://leetcode.com/submissions/detail/279789541/)

Solution-3：Another Iterative Solution

1.使用临时节点dummy，将其放在head的前面。

2.pre始终指向dummy，cur始终指向原始的head。

3.例如1->2->3->4->5->NULL，过程如下：
第一轮：2->1->3->4->5->NULL
第二轮：3->2->1->4->5->NULL
第三轮：4->3->2->1->5->NULL
第四轮：5->4->3->2->1->NULL

4.最后返回dummy.next(pre->next)即可。

时间复杂度：O(n)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/289498490/)

【链表专项】203.Remove Linked List Elements

发表于 2019-12-29

一、题意

题目链接: 203.Remove Linked List Elements

1.Description

【输入】
给定一个单链表，头为head。
给定一个整数val。
【输出】
在上述链表中，删除所有值为val的节点。

2.Example

Input: 1->2->6->3->4->5->6, val = 6
Output: 1->2->3->4->5

3.Corner Case

  1. 链表为NULL的时候，不做任何操作。
  2. 删除后，链表为空的时候，返回NULL。
  3. 删除后，链表不为空的时候，返回newHead。

二、题解

Solution-1

1.回忆【237.Delete Node in a Linked List】，给定一个目标节点node，通过将其后继节点赋值给node的办法，在链表中删除该节点node。

2.在本题中，给定了head，如果遍历到某个节点，发现其需要删除时，可以将其前一个节点和其后继节点接起来：
由于访问当前节点，需要使用到前一个节点。
所以可以对于每个节点，通过检查其next节点，如果其需要删除，则将其后继接在本节点的后面。

3.当链表头需要被删除时，可以使用dummyHead的方法：将一个临时节点dummy接在原有head的前面。最后直接返回dummy.next即可。

时间复杂度：O(n)
空间复杂度：O(1)

3.Code(https://leetcode.com/submissions/detail/280528928/)

【链表专项】237.Delete Node in a Linked List

发表于 2019-12-29

一、题意

题目链接: 237.Delete Node in a Linked List

1.Description

【输入】
给定一个单链表中的节点node（非链表尾）。
【输出】
在原链表中删除该目标节点node。

2.Example

Input: head = [4,5,1,9], node = 1
Output: [4,5,9]
Explanation: You are given the third node with value 1, the linked list should become 4 -> 5 -> 9 after calling your function.

3.Corner Case

  1. 目标节点node不是链表尾，且保证是链表中的合法节点。
  2. 单链表至少有两个元素。
  3. 所有节点的value不重复。
  4. 函数返回值为void。

二、题解

Solution-1

1.给定一个节点，在链表中删除掉该节点，但是并没有给定链表的head。

2.思路1：如果给定了head的话，就可以直接把node的前一个节点接到node的后一个节点，这样node就不在链表中了。

3.思路2：本题目并没有给定head指针。
所以：把node后面一个节点的值拷贝到node里面，然后删除掉node->next，达成同样的删除效果。

4.结构体变量直接赋值；不使用的节点记得释放内存；

时间复杂度：O(1)
空间复杂度：O(1)

5.Code(https://leetcode.com/submissions/detail/280512957/)

【二叉树专项】106.Construct Binary Tree from Inorder and Postorder Traversal

发表于 2019-12-27 更新于 2019-12-29

一、题意

题目链接: 106.Construct Binary Tree from Inorder and Postorder Traversal

1.Description

【输入】
给定两个二维数组，分别代表一棵二叉树的：
后序遍历序列
中序遍历序列
【输出】
求出这棵二叉树。

2.Example

Input:
inorder = [9,3,15,20,7]
postorder = [9,15,7,20,3]

Output: [3,9,20,NULL,NULL,15,7]

3.Corner Case

  1. 目标数组为空时，树为NULL。
  2. 目标数组长度为1时，树为单节点。

二、题解

Solution-1：Recursive Solution(Postorder)

1.根据以下特性：
postorder序列的最后一个数字一定是：root的值
inorder序列：root的值左边都是root的左子树，右边都是root的右子树。

2.对于任意任意两个序列：build(postStart, postEnd, inStart, inEnd)
a.根节点root一定是postorder[postEnd]
b.在inorder中找到root的位置，记为pos。则左子树的节点数目：pos - inStart
c.递归构建左子树：build(postStart, postStart + (pos - inStart) - 1, inStart, pos - 1)
d.递归构建右子树：build(postStart + (pos - inStart), postEnd - 1, pos + 1, inEnd)
e.将c.d两步的返回值赋值为root的左子树和右子树，return给上一层。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/278100841/)
