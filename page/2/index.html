【DP专项】516.Longest Palindromic Subsequence

发表于 2020-01-16

一、题意

题目链接: 516.Longest Palindromic Subsequence

1.Description

【输入】
给定一个长度为n的字符串s。
【输出】
找出一个s的一个子序列，该子序列构成回文串，且长度最大。

2.Example

Input: “bbbab”
Output: 4
One possible longest palindromic subsequence is “bbbb”.

Input: “cbbd”
Output: 2
One possible longest palindromic subsequence is “bb”.

3.Corner Case

  1. 当n <= 1时，答案为n；
  2. n <= 1000；

二、题解

思路

  1. 回忆【005.Longest Palindromic Substring】，我们找出任意一个s的最长回文子数组。
    而本题要求的是s的最长回文子序列的长度。

  2. 回忆【300.Longest Increasing Subsequence】中，有如下分析：
    【子序列】不要求连续，所以相较于【子数组】来说，数目更多。
    对于一个长度为n的数组来说：
    【子数组】（非空）的个数为：n * (n + 1) / 2；复杂度：O(n * n)
    【子序列】（非空）的个数为：(2 ^ n) - 1；复杂度：O(2 ^ n)

  3. 如果采用暴力方法的话，时间复杂度为O(n * (2 ^ n))；

Solution-1：Dynamic Programming

  1. Q: What is important so far??? A: Cur char(end); Start char(start); Pre length;

  2. 状态定义：dp[i][j]代表：在范围[i, j]内，最长的回文子序列的长度。答案即：dp[0][n - 1]；

  3. 边界条件：
    a. i == j时，dp[i][j] = 1；
    b. i > j时，dp[i][j]无意义；

  4. 转移方程：
    a. 若s[i] == s[j]，则dp[i][j] = 2 + dp[i + 1][j - 1]；
    考虑dp[i + 1][j - 1]的合法性：(i + 1 > j - 1时，dp[i + 1][j - 1]为0)
    b. 若s[i] != s[j]，则dp[i][j] = max(dp[i][j - 1], dp[i + 1][j])；
    不需要考虑dp[i][j - 1]和dp[i + 1][j]的合法性，画图：二维数组即可看出。

  5. 计算顺序：
    【Backward-DP】
    从上面的状态转移方程中可以看出：
    计算当前row的时候(i)，需要使用到下一row的内容(i + 1)。所以i从n - 1开始，到0结束；
    计算当前col的时候(j)，需要使用到前一col的内容(j - 1)。所以j从i开始，到n - 1结束；

  6. Code(https://leetcode.com/submissions/detail/294675718/)
    时间复杂度：O(n * n)
    空间复杂度：O(n * n)

  7. 空间优化：由于计算第(i)行的时候，只需要用到第(i + 1)行的数据，所以可以优化为O(n)的空间复杂度。
    具体过程：
    a. 在计算dp[i][j]的时候，需要用到它左下角的元素、左边的元素、下边的元素。
    b. 考虑dp数组压缩为一维的情况：左边的元素(dp[j - 1])、下边的元素(dp[j])。
    c. 由于在计算出来dp[j - 1]的时候，其之前的值已被覆盖。而被覆盖的该值，对于dp[j]来说，就是其左下角的元素。
    d. 所以我们在计算dp[j]的时候，先将其已存的值保存到prev变量中。在计算dp[j + 1]的时候，直接使用prev变量即可，计算后再更新prev变量。

  8. Code(https://leetcode.com/submissions/detail/294680816/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)

【DP专项】005.Longest Palindromic Substring

发表于 2020-01-08

一、题意

题目链接: 005.Longest Palindromic Substring

1.Description

【输入】
给定一个长度为n的字符串s；
【输出】
求出s的某个最长回文子串。可能存在多个，任意返回一个即可。

2.Example

Input: “babad”
Output: “bab”
Note: “aba” is also a valid answer.

3.Corner Case

  1. n <= 1000
  2. 若n为0，返回空串；
  3. 若n为1，返回s；

二、题解

思路

  1. 同样，可以用brute force，找出判断所有的子串，看其是否为回文串。
    找出最大的长度，以及开始的索引startIndex。

Solution-1：Dynamic Programming

  1. 状态定义：dp[i][j]表示：子字符串s[i]…s[j]是否为回文串，不是的话值为0，是的话值为其长度。

  2. Code(https://leetcode.com/submissions/detail/292341253/)
    时间复杂度：O(n * n)
    空间复杂度：O(n * n)

  3. Code(https://leetcode.com/submissions/detail/292065399/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)

Solution-2：中心扩展法

  1. 记录最长回文子串的长度；记录最长回文子串的起始索引；

  2. 有以下两种情况：
    当len1 > len2时，中心点为(i)，长度为len1，所以起始索引为：i - len1 / 2
    当len2 >= len1时，中心点为(i, i + 1)，长度为len2，所以起始索引为：i + 1 - len2 / 2

  3. Code(https://leetcode.com/submissions/detail/292349317/)
    时间复杂度：O(n * n)
    空间复杂度：O(1)

【DP专项】647.Palindromic Substrings

发表于 2020-01-08

一、题意

题目链接: 647.Palindromic Substrings

1.Description

【输入】
给定一个长度为n的字符串s；
【输出】
求出s的子串中，一共有多少个是回文串。
注意：若存在两个子串，其内容相同，但在s中的起始结束索引不一样。认为是两个不同的子串。

2.Example

Input: “abc”
Output: 3
Explanation: Three palindromic strings: “a”, “b”, “c”.

Input: “aaa”
Output: 6
Explanation: Six palindromic strings: “a”, “a”, “a”, “aa”, “aa”, “aaa”.

3.Corner Case

  1. n为0时，答案为0
  2. n为1时，答案为1

二、题解

Solution-1：Brute Force

  1. 回忆【300.Longest Increasing Subsequence】中，一个长度为n的数组的子数组的个数为：n * (n + 1) / 2。

  2. 在本题中，同样考虑s的所有子数组，并逐一判断其是否为回文串。

  3. Code(https://leetcode.com/submissions/detail/292045576/)
    时间复杂度：O(n ^ 3)
    空间复杂度：O(1)

Solution-2：Dynamic Programming

  1. Q: What is important so far??? A: Cur range([i, j]); s[i] and s[j]; Previous calculated range([i + 1, j - 1]);

  2. 状态定义：dp[i][j]代表：子串s[i]…s[j]是否为回文串。bool类型即可。i, j均小于n。

  3. 边界条件：
    a. 若i == j，则子串为单个字符组成，是回文串。
    b. 若j > i，dp[i][j] = false。

  4. 转移方程：dp[i][j]当且仅当以下三种情况为true，否则为false：
    a. i == j
    b. s[i] == s[j] && j == i + 1
    c. s[i] == s[j] && dp[i + 1][j - 1]

  5. 计算顺序：
    【Backward-DP】
    i作为子串的start：[0, n - 1]
    j作为子串的end：[i, n - 1]
    由于计算dp[i][j]时，需要考虑dp[i + 1][j - 1]。所以i从n - 1开始，到0结束。
    在固定好i开始后，可以计算所有的dp[i][j]了。j的遍历顺序：既可从n - 1开始，到i结束；也可从0开始，到n - 1结束。

  6. 空间优化：二维降低到一维。

  7. Code(https://leetcode.com/submissions/detail/292053671/)
    时间复杂度：O(n * n)
    空间复杂度：O(n * n)

  8. Code(https://leetcode.com/submissions/detail/292053439/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)

Solution-3：中心扩展法

  1. i在范围[0, n - 1]上遍历，每次遍历考虑s[i]；

  2. 以s[i]为中心，向两边扩展。找出以s[i]为中心的回文串的个数，累加起来。

  3. Code(https://leetcode.com/submissions/detail/292058937/)
    时间复杂度：O(n * n)
    空间复杂度：O(1)

【DP专项】139.Word Break

发表于 2020-01-07

一、题意

题目链接: 139.Word Break

1.Description

【输入】
给定一个长度为n的字符串s。s保证非空。
给定一个字符串数组wordDict。wordDict非空。代表着一个字典。
【输出】
判断：能否由这些字典中的若干个元素（子字符串），通过拼凑连接，组合成字符串s。

2.Example

Input: s = “applepenapple”, wordDict = [“apple”, “pen”]
Output: true

Input: s = “catsandog”, wordDict = [“cats”, “dog”, “sand”, “and”, “cat”]
Output: false

3.Corner Case

  1. 字典中元素不重复。
  2. 字典中的同一元素可使用多次。

二、题解

思路

  1. 回忆【322.Coin Change】中，同样也是：在一个数组中，选择若干个元素，使其sum等于target。
    解决方法是使用动态规划：将子问题的解保存下来，考虑当前问题的时候：遍历所有满足条件的子问题，找出一个最优解（最少硬币数）。

  2. 回忆【300.Longest Increasing Subsequence】，同样也是：求解当前的问题的时候，遍历所有满足条件的子问题，找出一个最优解（最大长度）。

  3. 在本题目中，求解的对象是字符串，而非一个数字。所以可以参考【300.LIS】：遍历每个位置，将每个位置构造成一个子问题。

Solution-1：Dynamic Programming

  1. 状态定义：dp[i]代表：以s[0]为开头，以s[i - 1]为结尾的子串，是否可以由字典中给定的元素拼接出来。bool类型即可。答案即dp[n]。

  2. 边界条件：dp[0] = true;

  3. 转移方程：dp[i] = true; 当且仅当存在一个j，使得dp[j] = true，且substr[j + 1, i]在字典中。

  4. 计算顺序：
    【Backward-DP】
    对于位置i，需要考虑其前面的情况。所以i从前到后遍历。
    依次计算dp[1]到dp[n]。本题答案即为dp[n]。
    考虑计算dp[i]，也就是：以s[0]为开头，以s[i - 1]为结尾的子串，是否可以通过字典中元素拼接形成。
    需要依次考虑以下【i次】：
    子串为: s[i - 1]，是否在字典中。与此同时dp[i - 1] = true；
    子串为: s[i - 2] s[i - 1]，是否在字典中。与此同时dp[i - 2] = true；
    …
    子串为：s[0] s[1] … s[i - 1]，是否在字典中。与此同时dp[0] = true；

  5. 空间优化：无法由一维线性降低到常数空间；

  6. 判断子字符串是否在字典中，可以使用set：
    a. std::set 是关联容器，含有 Key 类型对象的已排序集。用比较函数 比较 (Compare) 进行排序。搜索、移除和插入拥有对数复杂度。 set 通常以红黑树实现。
    b. unordered_set is 是含有 Key 类型唯一对象集合的关联容器。搜索、插入和移除拥有平均常数时间复杂度。

  7. Code(https://leetcode.com/submissions/detail/292002088/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)

  8. 时间优化：
    a. 统计出字典中元素的最大长度，记为maxLen。假设为5。
    b. 假设当前计算dp[8]，也就是：以s[0]为开头，以s[7]为结尾的子串，是否可以通过字典中元素拼接形成。
    按照上面的分析，需要依次考虑以下【8次】：
    子串为: s[7]，是否在字典中。与此同时dp[7] = true；
    子串为: s[6]s[7]，是否在字典中。与此同时dp[6] = true；
    子串为: s[5]s[6]s[7]，是否在字典中。与此同时dp[5] = true；
    子串为: s[4]s[5]s[6]s[7]，是否在字典中。与此同时dp[4] = true；
    子串为: s[3]s[4]s[5]s[6]s[7]，是否在字典中。与此同时dp[3] = true；

子串为: s[2]s[3]s[4]s[5]s[6]s[7]，是否在字典中。与此同时dp[2] = true；
子串为: s[1]s[2]s[3]s[4]s[5]s[6]s[7]，是否在字典中。与此同时dp[1] = true；
子串为：s[0]s[1]s[2]s[3]s[4]s[5]s[6]s[7]，是否在字典中。与此同时dp[0] = true；
c. 由于子串最大长度为5，所有上面8次遍历中，后面3次其实是冗余的，不需要考虑。
d. 当i大于maxLen时：j从(i - 1)开始，直到(i - maxLen)结束。
e. 当i小于等于maxLen的情况下，j从(i - 1)开始，直到0结束。

  1. Code(https://leetcode.com/submissions/detail/292004122/)
    时间复杂度：O(n * n)
    空间复杂度：O(n)

【DP专项】279.Perfect Squares

发表于 2020-01-07

一、题意

题目链接: 279.Perfect Squares

1.Description

【输入】
给定一个正整数n；
【输出】
求出至少需要多少个”完全平方数”，使得它们的和等于n；
完全平方数：1, 4, 9, 16,…

2.Example

Input: n = 12
Output: 3
Explanation: 12 = 4 + 4 + 4.

Input: n = 13
Output: 2
Explanation: 13 = 4 + 9.

3.Corner Case

  1. 可以重复使用同一个数；
  2. n保证大于0；

二、题解

思路

  1. 回忆【322.Coin Change】，从数组中选出一些元素，使得其和为target，求最少使用几个元素。【可选可不选】【可选多次】【求最小个数】

  2. 本题几乎完全一样，数组为：[1, 4, 9, 16…]。

Solution-1：Dynamic Programming

  1. Q: What is important so far??? A: Cur value; Potential pre value; Pre cnt;

  2. 状态定义：dp[i]代表：最少需要多少个完全平方数，使其sum为i。

  3. 边界条件：dp[1] = 1。dp[0] = 0。（当i本身就是完全平方数的时候，此时dp[i] = 1 + dp[0] = 1）

  4. 转移方程：dp[i] = min(1 + dp[i - j * j])。其中1 <= j * j <= i。

  5. 计算顺序：
    【Backward-DP】
    当计算dp[i]的时候，需要使用到dp[i - j * j]的值。而(i - j * j)一定不超过i，所以从前往后计算；

  6. 空间优化：无法由一维线性降至常数；

  7. Code(https://leetcode.com/submissions/detail/261940601/)
    时间复杂度：O(n * sqrt(n))
    空间复杂度：O(n)

Solution-2：Math

  1. 四平方和定理(Lagrange’s four-square theorem)说明每个正整数均可表示为4个整数的平方和。
    所以本题目的答案就在[1,2,3,4]范围内。

  2. 如果n本身就是完全平方数，则答案为1：由它自身一个元素组成；

  3. The result is 4 if and only if n can be written in the form of 4^k(8m + 7).
    Please refer to Legendre’s three-square theorem.

  4. 考虑答案为2的情况：即由两个完全平方数组成。

  5. 其余情况，答案只剩3了。

  6. Code(https://leetcode.com/submissions/detail/264023785/)
    时间复杂度：O(sqrt(n))
    空间复杂度：O(1)
