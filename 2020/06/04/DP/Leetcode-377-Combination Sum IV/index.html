Leetcode-377.Combination Sum IV

发表于 2020-06-04 更新于 2020-06-05 分类于 Leetcode ， 数据结构与算法

一、题意

黑板上排列组合，你舍得解开吗~~

0、题目链接

377.Combination Sum IV

1、Description

【输入】
给定一个大小为n的整数数组nums；
nums性质：元素都是正数；无重复元素；
给定一个整数target；
target性质：正数；
【输出】
求出有几种方案，使得从nums中拿出一些数字，其和为target。
【要求】
无；

2、Example

nums = [1, 2, 3]
target = 4

The possible combination ways are:
(1, 1, 1, 1)
(1, 3)
(3, 1)
(2, 2)
(1, 1, 2)
(1, 2, 1)
(2, 1, 1)

Note that different sequences are counted as different combinations.
Therefore the output is 7.

3、Corner Case

  1. target为正数；
  2. nums[i]为正数；
  3. nums中无重复数；
  4. n == 0时：而target是正数，return 0；
  5. 考虑n >= 1的情况；

二、题解

0、思考

  1. 从上面的例子中可以看出：

  2. 对于nums中的任意元素，可以认为其数量是不限的；

  3. 【相对顺序】是重要的，即：[211] != [112] != [121]；也就是求【Permutation】，而非【Combination】；

  4. 本题名为【Combination Sum IV】，但又考虑每个元素的【相对顺序】：可以改为【Permutation Sum】；

  5. 真正意义上的【Combination Sum】是另外一题：【518.Coin Change 2】；

1、Solution-1：Dynamic Programming

  1. 记dp[i]为：组成i的种类数。本题答案为：dp[target]。

  2. 假设此时，求dp[x]。且dp[0] ~ dp[x - 1]都已经求出来了。

  3. 要想组成x，且仅能使用nums中的n个数字：
    x = nums[0] + (x - nums[0])：
    x = nums[1] + (x - nums[1])：
    …
    x = nums[n - 1] + (x - nums[n - 1])：

  4. 若(x - nums[i]) >= 0的时候，dp[x] += dp[x - nums[i]]。

  5. 边界值：dp[0] = 1；
    当x == nums[i]的时候，x = nums[i] + 0。dp[x] += 1，满足。

  6. 计算方向：
    【Backward-DP】：计算dp[x]的时候，考虑dp[0] ~ dp[x - 1]的某些值。
    i遍历target，从1开始，到target结束。
    对于每个i：j从0开始，到n - 1结束。

  7. 注意：当用例为：[3,33,333]，10000时：通过观察，可以发现dp[10000] = 0。因为不管怎么组合，都不可能使得sum为10000。
    此时在从1计算到10000的时候，存在某些i，使得dp[i]非常大，使用long long也不足以表示其方案数。
    解决方法：int dp[]可以换成unsigned int dp[]。这样，当加法overflow的时候，unsigned int会自动处理溢出。而不会报Runtime Error。
    For example, unsigned int x = 4294967295 is ok if do x += 1 operation, x will be 0 without any runtime error.

  8. 在某些题中，也可能会要求：当加法overflow的时候，答案模一个值（如：10 0000 0007）；

  9. Code(https://leetcode.com/submissions/detail/348805577/)
    AC
    时间复杂度：O(target * n)
    空间复杂度：O(target)

  10. 举个例子：
    nums = [1, 5, 10, 25]，target = 11；
    i = 0: dp[0] = 1;
    i = 1: 考虑[1]; dp[1] = 1; [1 /]
    i = 2: 考虑[1]; dp[2] = 1; [1 1/]
    i = 3: 考虑[1]; dp[3] = 1; [1 11/]
    i = 4: 考虑[1]; dp[4] = 1; [1 111/]
    i = 5: 考虑[1][5]; dp[5] = 2; [1 1111/] [5 /]
    i = 6: 考虑[1][5]; dp[6] = 3; [1 11111/][1 5/] [5 1/]
    i = 7: 考虑[1][5]; dp[7] = 4; [1 111111/][1 15/][1 51/] [5 11/]
    i = 8: 考虑[1][5]; dp[8] = 5; [1 1111111/][1 115/][1 151/][1 511/] [5 111/]
    i = 9: 考虑[1][5]; dp[9] = 6; [1 11111111][1 1115][1 1151][1 1511][1 5111] [5 1111/]
    i = 10: 考虑[1][5][10]; dp[10] = 9; [1 111111111/][1 11115/][1 11151/][1 11511/][1 15111/][1 51111/] [5 11111/][5 5/] [10 /]
    i = 11: 考虑[1][5][10]; dp[11] = 13; 13 = dp[11 - 1] + dp[11 - 5] + dp[11 - 10] = 9 + 3 + 1 = 13
