Leetcode-343.Integer Break

发表于 2020-01-17 更新于 2020-04-23 分类于 Leetcode ， 数据结构与算法

一、题意

0、题目链接

343.Integer Break

1、Description

【输入】
给定一个正整数n；
【输出】
将这个正整数n，拆分成若干个正整数：使其sum为n，同时使其product最大。求出这个最大乘积；
【要求】
至少拆分成两个元素。

2、Example

Input: 10
Output: 36
Explanation: 10 = 3 + 3 + 4, 3 × 3 × 4 = 36.

3、Corner Case

  1. 2 <= n <= 58
  2. 若n为2，答案为1；(1 * 1)
  3. 若n为3，答案为2；(1 * 2)
  4. 若n为4，答案为4；(2 * 2)

二、题解

1、Solution-1：Dynamic Programming

  1. 状态定义：dp[i]代表：将i分解成至少两个正整数的序列，其中能产生的最大的乘积。

  2. 边界条件：dp[0] = dp[1] = -1(不可能分解); dp[2] = 1;

  3. 转移方程：dp[i] = max(dp[i], max(j, dp[j]) * max(i - j, dp[i - j]));

  4. 计算顺序：
    【Backward-DP】
    由于在计算dp[i]的时候，需要使用到dp[1]…dp[i - 1]的值。所以i从3开始，到n结束。
    对于某个固定的i：j从1开始，到i - 1结束。

  5. 优化1：
    对于某个固定的i：j不必从1开始，到i - 1结束。
    从1开始，到i / 2结束即可；
    例如i为7时，j从1开始，到3结束；
    例如i为8时，j从1开始，到4结束；

    时间复杂度：O(n * n)
    空间复杂度：O(n)

  7. 优化2：根据Solution-2-贪心方法的结论：对于一个数字i，仅需考虑分解因子为：2、3的情况。

2、Solution-2：Greedy

  1. 能够使用“贪心算法”完成的问题，严格上来说，是要证明这个问题具有“贪心选择”性质；
    但是如果一个问题，不能使用贪心算法，只需要举一个反例就可以了。

  2. 在本题中，我们尝试证明本题可以使用某个贪心性质，来找到最优解。

  3. 证明思路：从小规模的数扩展到较大规模的数，发现其中的规律。

  4. 过程：
    a. 当n有分解因子1的时候：1 * (n - 1) < n，当n为任意数的时候满足。即要想得到最优解，一定不能有1作为分解因子；
    b. 当n有分解因子2的时候：2 * (n - 2) > n，当n > 4的时候满足。即当n > 4的时候，分解出2这个因子，与剩下的数(n - 2)得到的乘积，就肯定超过n了；
    c. 当n有分解因子3的时候：3 * (n - 3) > n，当n > 4.5的时候满足。即当n > 4.5的时候，分解出3这个因子，与剩下的数(n - 3)得到的乘积，就肯定超过n了。
    d. 当n有分解因子4的时候：分解出4与(n - 4)的乘积等价于分解出2、2与(n - 4)的乘积，因此没有必要考虑分解出正整数加法因子4。
    e. 当n有分解因子5的时候：分解出5与(n - 5)的乘积还不如分解出2、3与(n - 5)的乘积，因此没有必要考虑分解出正整数加法因子5。
    f. 当n有分解因子6的时候：分解出6与(n - 6)的乘积还不如分解出3、3与(n - 6)的乘积，因此没有必要考虑分解出正整数加法因子6。
    g. 结论1：大于等于4的正整数加法因子i都不必考虑了，因为分解成i * (n - i)，都小于等于分解出合适数量的2和3相乘以后，再和(n - i)的乘积。
    h. 结论2：当n >= 5的时候，优先考虑分解成3，而非2。此时：3 * (n - 3) >= 2 * (n - 2)
    i. 所以：我们应该尽可能分解出3，直到最后剩下4或者2。
    j. 所以：综上所述，当n <= 4的时候，专门计算一下就可以了；在n大于4的前提下，尽可能分解出3，把剩余的数字累乘起来就行；
