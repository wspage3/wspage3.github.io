【DP专项】338.Counting Bits

一、题意

题目链接: 338.Counting Bits

1.Description

【输入】
给定一个非负整数n。
【输出】
求出范围[0, n]内，所有元素的二进制形式中1的个数。以数组的形式返回（数组长度为：n + 1）。

2.Example

Input: 5
0: 0000
1: 0001
2: 0010
3: 0011
4: 0100
5: 0101
Output: [0,1,1,2,1,2]

3.Corner Case

  1. 当n为0时，返回长度为1的数组。[0]
  2. 要求空间复杂度O(n)
  3. 要求时间复杂度O(n)，而非O(n * l)，其中l是整数的比特数。4字节时，l = 32。
  4. 要求不适用内置函数，如：__builtin_popcount(n)

二、题解

Solution-1：Brute Force

  1. 依次遍历[0, n]，单独求出每个数字的二进制表示方式中1的个数。

  2. 对于任一数字，让其不断除以2，计算最低位1的个数。

  3. Code(https://leetcode.com/submissions/detail/282685606/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

  4. Code(https://leetcode.com/submissions/detail/282686107/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

  5. Code(https://leetcode.com/submissions/detail/282686317/)
    时间复杂度：O(n * l)
    空间复杂度：O(n) //结果数组

Solution-2：Dynamic Programming

  1. 考虑1100000000001这个数字，其十进制为：6145。
    按照Solution-1，我们对这个数字每次除以2（右移一位），需要处理13次。
    而其中有10个0，3个1。有没有办法跳过这些0呢？

  2. temp不再右移一位。而是：temp = temp & (temp - 1);
    继续上面的例子：
    1100000000001：计数一次
    1100000000000：计数一次
    1000000000000：计数一次
    0000000000000：结束
    可以看到，我们只处理了3次，也就是说通过这样的方法，对于任意数字x。可以将时间复杂度降低为O(k)，k是其中1的个数。

  3. Code(https://leetcode.com/submissions/detail/282687122/)
    时间复杂度：O(n * k) //k是数字的二进制表示形式下，其中1的个数。
    空间复杂度：O(n)

Solution-3：Dynamic Programming

  1. 根据Solution-2的启示：我们可以发现：res[i] = res[i & (i - 1)] + 1;

    空间复杂度：O(n)
