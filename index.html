【二叉树专项】100.Same Tree

一、题意

题目链接: 100.Same Tree

1.Description

【输入】
给定两棵二叉树。
【输出】
判断这两棵二叉树是否是同一个。

2.Example

Input: [1,2], [1,null,2]
Output:
false

3.Corner Case

  1. 两树都为空时，true。
  2. 两树其中一个为空时，false。
  3. 然后考虑两树都不为空的场景。

二、题解

二叉树节点的定义：

Solution-1：Recursive solution(DFS)

1.当两树p, q都不为空的情况下：
首先，判断节点q和节点q的value是否相等，不相等则return false。
其次，递归判断p和q的左子树是否same，以及p和q的右子树是否same。

时间复杂度：O(n)
空间复杂度：O(n) //最坏n（完全不平衡二叉树），最好log n（完全平衡二叉树）

本质顺序：先序遍历的recursive版本。

2.同样的,也可以使用iterative + stack的版本来对两棵树同时进行前序/中序/后序遍历.
遍历的过程中进行比较.

3.Code(https://leetcode.com/submissions/detail/275771314/)

Solution-2：Iterative solution using Queue(BFS)

1.按照bfs的顺序,将p和q所有的节点入队列.
每次取出一个元素进行比较.
如果比较通过,则将它们的左右儿子同时放入各自的队列中.

2.若两个树same的话,中途不会return false.且最终两个队列里都为空.

本质:层次遍历的iterative版本

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275800085/)

【二叉树专项】103.Binary Tree Zigzag Level Order Traversal

一、题意

题目链接: 103.Binary Tree Zigzag Level Order Traversal

1.Description

【输入】
给定一棵二叉树。
【输出】
求出这棵二叉树的Zigzag的level序遍历结果，并以二维数组的形式返回。
垂直顺序：从根往叶。
水平顺序：先从左往右，后从右往左。交替进行。

2.Example

Input: [3,9,20,null,null,15,7]
Output:
[
[3],
[20,9],
[15,7]
]

3.Corner Case

  1. 树为空时，返回空数组。

二、题解

二叉树节点的定义：

Solution-1：Recursive solution + Reverse

1.和上题相比，垂直顺序不变。
水平顺序变成：依次从左往右，从右往左。
将遍历出来的ans[i]进行Reverse。i = 1，3，5…

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/276450407/)

Solution-2：Iterative solution using Queue + Reverse

1.当计算出res[i]，准备将其放入ans的时候，检查：若i是奇数，则reverse。
2.当一轮中往res[i]中添加同一层的元素时，检查：若i时奇数，则sub[n - 1 - i] = node->val。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276462285/)

Solution-3：Iterative solution using Deque

1./* zig（正常）：push_back, front, pop_front, 先left后right /
/ zag（反序）：push_front, back, pop_back, 先right后left */

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/276467788/)

【二叉树专项】102.Binary Tree Level Order Traversal

一、题意

题目链接: 102.Binary Tree Level Order Traversal

1.Description

【输入】
给定一棵二叉树。
【输出】
求出这棵二叉树的level序遍历结果，并以二维数组的形式返回。
从根往叶，从左往右。

2.Example

Input: [3,9,20,null,null,15,7]
Output:
[
[3],
[9,20],
[15,7]
]

3.Corner Case

  1. 树为空时，返回空数组。

二、题解

二叉树节点的定义：

Solution-1：Recursive solution

0.前序、中序、后序：都属于dfs。当实现迭代版本时，我们使用的都是stack。
1.层次遍历：依旧用dfs实现。

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/275764307/)

Solution-2：Iterative solution using Queue

0.前序、中序、后序：都属于dfs。当实现迭代版本时，我们使用的都是Stack模拟。
1.层次遍历：实现迭代版本时，使用bfs，用Queue模拟。
2.每一轮，从队列中依次拿出n个元素，这些元素一定是同一层的。将这个结果放入ans中。
拿的时候，同时考虑其左右儿子，如果有的话，将其加入队列中。
3.直到队列为空的时候，所有的元素都被考虑过了。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/275767633/)

【二叉树专项】145.Binary Tree Postorder Traversal

一、题意

题目链接: 145.Binary Tree Postorder Traversal

1.Description

【输入】
给定一棵二叉树。
【输出】
求出这棵二叉树的后序遍历结果，并以数组的形式返回。

2.Example

Input: [1,null,2,3]
Output: [3,2,1]

3.Corner Case

  1. 树为空时，返回空数组。

二、题解

二叉树节点的定义：

Solution-1：Recursive solution

1.后序遍历：左子树 -> 右子树 -> root

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/274963826/)

Solution-2：Iterative solution using Stack

1.后序遍历：左子树 -> 右子树 -> root

2.思路：
后序遍历：左子树 -> 右子树 -> root
前序遍历：root -> 左子树 -> 右子树
可以在前序遍历的基础上，做点改动，得到后序遍历的列表：
在preorder的iterative版本，我们是在出root的时候，先放进去right，再放进去left。使得先访问left，再访问right。
现在我们调换一下这个顺序，使得访问顺序为：root -> 右子树 -> 左子树
最后再把结果反转一下即可。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/274966540/)

【二叉树专项】094.Binary Tree Inorder Traversal

一、题意

题目链接: 094.Binary Tree Inorder Traversal

1.Description

【输入】
给定一棵二叉树。
【输出】
求出这棵二叉树的中序遍历结果，并以数组的形式返回。

2.Example

Input: [1,null,2,3]
Output: [1,3,2]

3.Corner Case

  1. 树为空时，返回空数组。

二、题解

二叉树节点的定义：

Solution-1：Recursive solution

1.中序遍历：左子树 -> root -> 右子树

时间复杂度：O(n)
空间复杂度：O(n)

2.Code(https://leetcode.com/submissions/detail/274967419/)

Solution-2：Iterative solution using Stack

1.中序遍历：左子树 -> root -> 右子树

2.思路：
每到一个节点 A，因为根的访问在中间，将 A 入栈。
然后遍历左子树，
接着访问 A，
最后遍历右子树。

在访问完 A 后，A 就可以出栈了。因为 A 和其左子树都已经访问完成。

时间复杂度：O(n)
空间复杂度：O(n)

3.Code(https://leetcode.com/submissions/detail/274978572/)

一、题意

题目链接: 210.Course Schedule II

1.Description

【背景】
在某学期，有n门课程需要学习，编号为：0 ~ n - 1。
某些课程有依赖关系，例如：学习C++前必须先把C语言学习了。
如果course 0学习前必须要先学习course 1的话，记为：[0, 1]。
【输入】
给定数字n代表课程数目。
给定二维数组prerequisite代表所有的依赖关系。
【输出】
判断能否把所有的课程学习完成。
若可以，给出一个可能的顺序（可能的顺序可能不止一个，给出一个即可）。

2.Example

Input: 4, [[1,0],[2,0],[3,1],[3,2]]
Output: [0,1,2,3] or [0,2,1,3]
Explanation: There are a total of 4 courses to take. To take course 3 you should have finished both
courses 1 and 2. Both courses 1 and 2 should be taken after you finished course 0.
So one correct course order is [0,1,2,3]. Another correct ordering is [0,2,1,3] .

3.Corner Case

  1. 考虑n的合法性。
  2. 考虑cntEdges的合法性。

二、题解

分析

参考Leetcode-207

Solution-1：Topological Sort(Kahn’s Algorithm)(BFS)

1.出队列的顺序即为一个有效的拓扑顺序。
2.在return的时候判断ans中是否包含了所有的点即可。否则，说明有环，返回空数组。

时间复杂度：O(n + V) //n是节点个数，V是边的条数
空间复杂度：O(n ^ 2)

207.Course Schedule

一、题意

题目链接: 207.Course Schedule

1.Description

【背景】
在某学期，有n门课程需要学习，编号为：0 ~ n - 1。
某些课程有依赖关系，例如：学习C++前必须先把C语言学习了。
如果course 0学习前必须要先学习course 1的话，记为：[0, 1]。
【输入】
给定数字n代表课程数目。
给定二维数组prerequisite代表所有的依赖关系。
【输出】
判断能否把所有的课程学习完成。

2.Example

Input: 2, [[1,0],[0,1]]
Output: false
Explanation: There are a total of 2 courses to take.
To take course 1 you should have finished course 0, and to take course 0 you should also have finished course 1. So it is impossible.

3.Corner Case

  1. 判断n的合法性
  2. 判断cntEdges的合法性

二、题解

分析

本题目实际上是：求一张有向图中是否有环。
n代表：该有向图中有n个节点。
prerequisite代表：该有向图中所有的边。

Solution-1：Topological Sort(Kahn’s Algorithm)(BFS)

1.Kahn’s Algorithm本质上是一个基于入度的拓扑排序。
具体步骤是，我们每次去掉入度为0的点，将该点加入拓扑排序，
同时删去与其连接的边（其它节点的入度会受到影响），
直到去掉所有的点为止，
如果中途遇到不存在入度为0的点的情况，那么，就认为这个有向图不是拓扑排序的。

2.具体过程如下：
首先，根据输入，把图存起来，此处使用二维vector作为邻接表。（还可以使用二维vector作为邻接矩阵）。
同时，存储图的时候，计算每个节点的入度indegree。
然后，将图中所有入度为0的点依次进入队列。
最后，每次从队首拿出一个元素来，将其放入拓扑排序的结果中。将其入度置为-1。
同时，遍历其后继节点，将它们的入度都减一。如果减一后为0，则继续入队。
当所有点都考虑完成后，拓扑排序的结果应该包含所有点。若不包含某些点，说明该有向图中有环存在。

时间复杂度：O(n + V) //n是节点个数，V是边的条数
空间复杂度：O(n ^ 2)

3.Code(https://leetcode.com/submissions/detail/287858657/)

Solution-2：DFS

1.采用DFS的路径，从每个节点一直往下走，将走过的节点标记。
若中途又经过了本节点，说明一定存在环。

2.具体过程如下：
首先，先将图存起来，这次我们用邻接矩阵存储。
同时，开一个大小为n的数组，用于标记每一个节点。
然后，从第一个节点开始进行dfs，依次考察所有n个节点。若某个节点处发现了环，则直接return false。
在某个节点i处dfs过程：
首先判断节点i在之前是否已经判断通过了，如果通过了，则return true。
其次判断节点i是否在本轮中正在考察，如果正在考察的同时，又遍历到了自己，说明一定有环。
最后：
a.将visited[i] = 1;
b.以其所有后继节点为起点，依次考察。
c.如果都通过了，则将visited[i] = -1;代表以i为起点的dfs路径，没有发现环。return true即可。

时间复杂度：O(n + V)
空间复杂度：O(n ^ 2)

3.Code(https://leetcode.com/submissions/detail/287867955/)

88.Merge Sorted Array

一、题意

题目链接: 88.Merge Sorted Array

1.Description

【输入】
给定两个整数数组nums1和nums2。两个数组都是有序的。
同时给定两个数字m，n。
m代表：nums1中前m个数字是有序的。
n代表：nums2中前n个数字是有序的。nums2的长度为n。
同时nums1的数组长度 >= m + n
【输出】
将nums2数组和nums1数组合并成一个有序的数组，且存在在nums1数组中。

2.Example

Input:
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6], n = 3

Output: [1,2,2,3,5,6]

3.Corner Case

  1. 若n = 0，不需要任何操作
  2. 若m = 0，将nums2拷贝到nums1中即可。

二、题解

Solution-1：Three Pointers

1.使用三个指针p1, p2, cur
cur指向nums1数组的末尾
p1指向nums1的最后一个有序元素
p2指向nums2的最后一个有序元素
当p1和p2均大于等于0时，选出一个较大的元素，放在nums1数组的末尾。

2.判断此时是p1小于0，还是p2小于0。
若p1小于0，说明nums1中所有的有序元素都已经考虑完成了。此时只要将nums2中未考虑的元素依次放入nums1中即可。
若p2小于0，说明nums2中所有的有序元素都已经考虑完成了。此时不需要任何操作，nums1中所有元素都是有序的了。

时间复杂度：O(m + n)
空间复杂度：O(1)

6.Code(https://leetcode.com/submissions/detail/287061095/)
